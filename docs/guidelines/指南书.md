## 2. 安装CUDA 13.0.2

需要安装最新CUDA以确保PyTorch 2.x + Flash Attention 2稳定运行。请从NVIDIA官方存档下载13.2。

下载页面：<https://developer.nvidia.com/cuda-13-2-0-download-archivee>

### 2.1 Windows安装

1. 在上述页面选择 **Operating System: Windows → Architecture: x86_64 → Version: 11/10 → Installer Type: exe (local)**。
2. 运行下载的`cuda_13.2_windows.exe` → 选择"Express（推荐）"安装。
3. 安装完成后在PowerShell中验证：

   ```powershell
   nvidia-smi
   nvcc --version
   ```

4. 若`nvcc`无法识别，请在系统环境变量`Path`中添加：

   ```
   C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v13.2\bin
   C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v13.2\libnvvp
   ```

5. 重启后再次确认`nvcc --version`。

> **驱动注意**：CUDA 13.x需要NVIDIA驱动580或以上。若为旧版驱动，请先通过GeForce Experience或NVIDIA下载中心进行更新。


## 3. Python环境与仓库准备

本项目使用[`uv`](https://github.com/astral-sh/uv)管理依赖，Python版本为3.13。

### 3.0 一行命令安装（最简单，无需git）✅

推荐初学者使用此方法。将下面这一行粘贴到PowerShell中——它会在`uv`缺失时自动安装，下载最新release，执行`uv sync`，并在**桌面创建"Anima LoRA GUI"快捷方式**：

```powershell
irm https://raw.githubusercontent.com/sorryhyun/anima_lora/main/install.ps1 | iex
```

安装到`.\anima_lora\`文件夹（用`$env:ANIMA_DIR`更改路径，用`$env:ANIMA_VERSION='v1.4.0'`锁定特定版本）。完成后：

1. `cd anima_lora`
2. 按**§4**创建Hugging Face令牌并用`hf auth login`登录
3. 用`python tasks.py download-models`下载模型
4. 双击桌面的**"Anima LoRA GUI"**快捷方式启动（或`python tasks.py gui`）

> 此方法面向以GUI为主的使用方式，**不会**安装`make`。大多数操作都是GUI按钮，因此没有问题——但如果你想在CLI中运行`make ...`，请执行§3.3的`winget install ezwinports.make`，或用`python tasks.py`代替`make`。
>
> 之后在文件夹内用`python tasks.py update`更新（release tarball合并，无需git）。

下面的**§3.1–§3.3**是为偏好`git clone`或想了解每一步的用户准备的手动安装方法。

### 3.1 安装`uv`

  ```powershell
  irm https://astral.sh/uv/install.ps1 | iex
  ```

安装完成后打开新终端，确认`uv --version`有输出。

### 3.2 克隆仓库

```bash
git clone https://github.com/sorryhyun/anima_lora.git
cd anima_lora
```

> 本指南中所有路径均以`anima_lora/`目录为基准。所有命令请在该目录下执行。

### 3.3 安装依赖

```bash
winget install ezwinports.make
uv sync
```

`uv sync`会读取`pyproject.toml`/`uv.lock`，然后**在`anima_lora/`内创建一个`.venv/`文件夹（一个独立的 Python 环境）**，并把所有依赖装进其中。它不会改动你的系统 Python，因此不会污染其他项目——这是正常现象，你无需自己执行`pip install`。

> **这是新手最容易卡住的地方。** `uv sync`完成后，依赖包位于`.venv/`中，而不是全局 Python 里。如果你直接打开终端运行`python tasks.py ...`或`make lora`，该 shell 仍在使用*系统*的 Python，因缺少依赖而很容易报`ModuleNotFoundError`。你需要让命令指向`.venv/`里的 Python，有两种方式：

**方式 A —— `uv run`（无需激活，处处可用）。** 在任意命令前加上`uv run`，它会自动使用`.venv/`：

```bash
uv run make lora              # 或：uv run python tasks.py lora
uv run hf auth login
uv run make download-models
```

它不依赖 shell 状态，因此绝不会“忘记”环境，是最稳妥的方式。

**方式 B —— 每个终端激活一次 venv。** 激活后，在该 shell 会话期间`.venv/`的`python`会排在`PATH`最前面，于是无需`uv run`前缀，`make ...` / `python tasks.py ...`即可直接运行：

```powershell
.venv\Scripts\activate        # Windows (PowerShell / cmd)
```
```bash
source .venv/bin/activate     # Linux / macOS / WSL
```

当提示符出现`(anima_lora)`（或`(.venv)`）前缀时即表示成功。**每开一个新终端窗口都需要重新激活**——激活状态不会保留。退出请运行`deactivate`。VSCode 用户只需选择一次`.venv`解释器（命令面板 → *Python: Select Interpreter*），其集成终端便会自动激活。

> 本指南其余命令均写作普通的`make ...` / `python tasks.py ...`，前提是你**要么**已激活 venv（方式 B），**要么**加了`uv run`前缀（方式 A）。若命令以`ModuleNotFoundError`或`command not found: make`失败，几乎都是此原因——请激活 venv 或加上`uv run`。

2. 在终端中登录：


该命令会自动下载以下三项内容并整理到`models/`目录：

## 5. 数据集准备

Anima LoRA使用*图像 + 同名`.txt`描述词文件*的结构。以下是`image_dataset/`目录的示例：

```
image_dataset/
├─ 00001.png
├─ 00001.txt
├─ 00002.jpg
├─ 00002.txt
├─ subfolder/
│  ├─ 00010.webp
│  └─ 00010.txt
└─ ...
```

### 5.1 描述词编写提示

- 按照Anima官方指南，标签顺序始终为 [meta] [character] [series] [artist] [general]。例如：

```
absurdres, safe, 1girl, chitanda eru, hyouka, @channel (caststation), full body, serafuku, She is saying hi.
```

- 根据个人实验，absurdres、highres、masterpiece等质量标签不写或保留最少即可，效果最佳。或者等正式发布的mod guidance出来后，完全不写质量标签也可以。
- 原始图像存放于`image_dataset/`（文件名自由，保持此位置）。

### 5.2 `num_repeats`是什么，何时需要调整？（摘要：**不需要调整**）

在`configs/base.toml`的`[[datasets.subsets]]`中可以看到`num_repeats = 1`。该值指定**每个epoch中每张图像重复使用的次数**，是kohya-ss风格的选项，在其他LoRA训练器指南中经常出现。

- **本指南标准工作流中保持`1`不变。** 将所有图像集中在`image_dataset/`一个目录进行训练的常见模式下，增大`num_repeats`只会*延长单个epoch*，效果等同于增加`max_train_epochs`。通过epoch数调整训练量更为直观，本项目所有预设和方法配置均以`num_repeats = 1`为前提进行调优。
- **何时增大才有意义？** 当一次训练中有*多个不同的子集（目录）*且目录间图像数量差异较大时，可作为*均衡工具*来提高小目录的出现频率（例如：角色A 1000张 + 角色B 50张，则仅B子集设置`num_repeats = 20`）。单目录训练不适用此场景。
- **在哪里修改？** `num_repeats`是数据集设置而非方法设置，因此不在`configs/methods/`、`configs/gui-methods/`或GUI训练标签页中暴露。如确实需要，请直接编辑`configs/base.toml`的`[[datasets.subsets]]`（或`--dataset_config <path>`指定的独立TOML）。*如果只是想对相同图像进行更多训练*，请增加`max_train_epochs`而非`num_repeats`。

## 6. 预处理：缩放·潜变量·文本嵌入缓存

为优化训练速度和VRAM占用，需预先执行**缩放 → VAE潜变量缓存 → 文本嵌入缓存**三个步骤。

```bash
make preprocess              # 同时执行三个步骤（LoRA / 普通训练用）
# 或分步执行
make preprocess-resize       # 1) image_dataset/ → post_image_dataset/resized/
make preprocess-vae          # 2) VAE潜变量缓存 → post_image_dataset/lora/
make preprocess-te           # 3) 文本编码器输出缓存 → post_image_dataset/lora/
make preprocess-pe           # （可选）PE-Core视觉编码器特征缓存 — 仅IP-Adapter使用
```

> **⚠️ 缓存会被复用 — 绝不会自动删除。**
> `make preprocess`（以及GUI的*Preprocess*按钮）会**直接复用已有缓存**。`post_image_dataset/lora/`中的`.npz` / `_te.safetensors` / `_pe.safetensors`文件*不会被覆盖或删除*，只会处理缺失的部分。因此重复执行速度很快，中途中断也是安全的。
>
> 换言之，在已有缓存的情况下再次运行`make preprocess`不会丢失现有数据。反之，如果**修改了描述词/分词器/缩放选项并需要从头重新生成**，则需要手动删除缓存目录（`post_image_dataset/lora/`或`post_image_dataset/easycontrol/`）后再执行。

### 6.1 缩放的作用

- 将图像缩放至VAE所需的像素对齐格式
- 自动分类到*(H/16) × (W/16) = 4032或4200个patch*两种token数系列的固定token桶（每个桶恰好填满其对应的token数）
- 自动排除过小的图像（默认0.5MP以下）并输出报告
- 结果以PNG格式保存至`post_image_dataset/resized/`

### 6.2 潜变量缓存

- 对所有缩放后的图像只运行一次VAE，将结果保存到磁盘
- 训练时无需将VAE加载到GPU，大幅节省VRAM
- 缓存位置：`post_image_dataset/lora/{stem}_{WxH}_anima.npz`
- 脚本：`scripts/preprocess/cache_latents.py`

### 6.3 文本嵌入缓存

- 预先计算Qwen3 0.6B + LLM适配器的输出
- 若`use_shuffled_caption_variants = true`，则同时缓存描述词逗号打乱变体（训练时随机选取）
- 缓存位置：`post_image_dataset/lora/{stem}_anima_te.safetensors`
- 描述词始终从`image_dataset/`的原始`.txt`文件读取（不复制到resized目录）
- 脚本：`scripts/preprocess/cache_text_embeddings.py`

### 6.4 PE视觉特征缓存（可选）

- 仅在 IP-Adapter 训练（及 DCW v4 fusion head）时需要
- 预先计算PE-Core-L14-336视觉编码器输出，使训练时无需加载视觉编码器
- 缓存位置：`post_image_dataset/lora/{stem}_anima_pe.safetensors`

> **何时需要重新生成缓存？**
> - 添加了新图像 → 直接再次执行`make preprocess`即可（现有缓存保留，只处理新内容）。
> - **修改了描述词**或**更改了分词器/填充相关选项** → 按上方⚠️说明，手动删除缓存目录（`post_image_dataset/lora/`）后再执行。简单重复执行只会*复用现有缓存*，变更不会生效。

## 7. GUI使用方法

基于PySide6的GUI提供配置编辑、数据集浏览、预处理、训练启动/监控、LoRA合并等功能，一体化操作。

```bash
make gui
make gui-shortcut   # （可选）在Windows桌面创建无控制台窗口的.lnk快捷方式
```

GUI主要标签页：

- **训练配置（Training Config）**：通过下拉菜单选择LoRA系列变体（推荐：`tlora` — Ortho + T-LoRA / 其他如`lora`、`tlora-8gb`、`tlora_ortho_reft`、`hydralora`、`reft`等），直接修改`presets.toml`预设（default / low_vram等）及所有训练参数后启动训练
- **预处理（Preprocess）**：一次性完成缩放 + VAE + 文本嵌入缓存
- **数据集（Dataset）**：预览图像/描述词并直接编辑描述词
- **合并（Merge）**：将训练好的LoRA烘焙到基础DiT中，生成ComfyUI独立检查点（仅支持基础LoRA / OrthoLoRA / T-LoRA）

GUI训练内部同样调用`train.py`，因此相同参数可在CLI中完整重现。GUI读取`configs/gui-methods/<variant>.toml`（每个变体对应独立文件，无toggle块），GUI中的变体列表与CLI的`make lora-gui GUI_PRESETS=<variant>`结果一致。当前变体列表可用`ls configs/gui-methods/`查看。

### 7.1 表单编辑与保存行为

训练/预处理子进程会重新读取磁盘上的variant TOML文件，因此只编辑表单而不保存的话，改动不会反映到训练中。GUI通过以下两种方式处理：

- **变更检测**：表单中任意字段（或`+ Extra args`文本框）被编辑后，`Save`按钮会变为橙色并显示`Save *`标记，表示*磁盘上的variant文件与当前显示不同*。按下`Save`或重新选择variant从磁盘加载后标记消失。
- **自动保存**：若忘记保存直接点击`Train` / `Preprocess`，系统会在子进程启动前自动将当前表单值写入variant文件。即屏幕上显示的值就是训练使用的值。（`Test`使用最近训练的检查点进行推理，不在自动保存范围内。）

> 如果只是试验性地修改某些参数，可以编辑表单后切换到其他variant再切回来 — 系统会从磁盘重新加载，编辑内容被丢弃。

### 7.2 自动恢复（checkpointing_epochs）

训练中途中断后再次点击`Train`，**系统会自动从最后保存的断点继续训练**。这是应对停电、OOM、误关窗口等情况的最实用功能之一，默认已开启。

GUI操作说明：

- 训练配置标签页的**Training组**中有`checkpointing_epochs`字段（gui-methods变体默认值`2`，`methods/lora.toml`默认值`4`）。每N个epoch保存一次恢复状态，覆盖同一文件，不会占用额外磁盘空间。
- 训练中断后用相同variant再次点击`Train`，日志中出现`auto-resuming from checkpoint at step N`即为正常恢复。无需手动设置任何标志。
- 训练正常结束后，恢复用临时文件会自动删除，最终结果保存为`output/ckpt/<output_name>.safetensors`。
- **更改数据集或核心配置（rank/LR/epoch数等）后**重新训练，请手动删除`output/ckpt/<output_name>-checkpoint-state/`目录后再点击`Train`，否则会在旧状态基础上继续训练。

详细行为参见[§8.6 自动恢复](#86-自动恢复-checkpointing_epochs)——其中也说明了与`save_every_n_epochs`的区别。

### 7.3 停止训练与关闭 GUI

训练并不在 GUI 窗口*内部*运行——当你点击`Train`时，任务会交给一个小型后台**训练守护进程（daemon）**，由它将`train.py`作为分离（detached）进程运行。这带来两个实际结果：

- **`Stop`按钮会中止当前的训练任务。** 守护进程会继续运行，并推进到队列中的下一个任务（若有），因此停止某个任务绝不会拆掉整个队列。这等同于 CLI 的`make daemon-kill`。（`Stop`也会取消正在进行的`Test`或`Preprocess`，这两者*确实*在 GUI 内部运行。）
- **关闭 GUI 并不会停止训练。** 由于任务在分离的守护进程中运行，关闭窗口后训练仍在继续——非常适合想通宵跑的长时间训练。重新打开 GUI（`make gui`）时，它会自动重新连接到仍在运行的任务，你会在日志中看到`Re-attached to running job …`，进度条与输出也会实时续上。（从 CLI 或 ComfyUI 训练节点启动的任务同样会这样显现。）

> 若要彻底关闭训练——既杀掉正在进行的任务，*又*停止守护进程以释放 GPU——请在 CLI 使用`make daemon-terminate`。仅靠`Stop`会让守护进程继续驻留。
>
> `Test`和`Preprocess`是例外：它们作为窗口内的子进程运行，因此关闭 GUI 会取消它们。

## 8. 运行训练

所有训练均通过TOML配置文件和HuggingFace Accelerate执行。配置合并顺序为`configs/base.toml → configs/presets.toml[<preset>] → configs/methods/<method>.toml → CLI参数`，方法配置优先级高于预设配置。

### 8.1 快速开始

**最推荐的起点是OrthoLoRA + T-LoRA（即`tlora`变体）**。这是稳定性、细节表现和风格保留最为均衡的组合，适合普通角色/风格LoRA训练直接使用。

```bash
# 推荐：Ortho + T-LoRA (gui-methods/tlora.toml)
make lora-gui GUI_PRESETS=tlora                  # 标准环境
PRESET=low_vram make lora-gui GUI_PRESETS=tlora-8gb   # VRAM 8~12GB

# 其他变体 (configs/gui-methods/<variant>.toml — 无toggle块的独立文件)
make lora-gui GUI_PRESETS=lora                   # 最简单的基础LoRA
make lora-gui GUI_PRESETS=tlora_ortho_reft       # Ortho + T-LoRA + ReFT叠加
make lora-gui GUI_PRESETS=hydralora              # MoE多头路由
make lora-gui GUI_PRESETS=reft                   # 单独ReFT

# Toggle块方式 (直接在configs/methods/lora.toml中选择变体)
make lora                          # presets.toml[default]
PRESET=low_vram make lora          # presets.toml[low_vram] — VRAM 8~12GB
PRESET=half make lora              # 使用数据集的一半快速实验
```

> **在CLI中直接覆盖参数**：可以像`make lora -- --network_dim 32 --max_train_epochs 24`这样直接传递额外参数（`tasks.py`同理）。

### 8.2 蒙版损失（排除文字气泡）

在漫画/漫画风格数据中排除*文字气泡或文字区域*的训练损失，可显著改善训练效果。

```bash
make mask          # SAM3 + MIT（在临时目录运行）→ post_image_dataset/masks/
make mask-clean    # 删除post_image_dataset/masks/
```

输出的PNG为黑白图像：**白色(255) = 训练对象**，**黑色(0) = 排除**。数据集子集有`post_image_dataset/masks/`时自动使用，否则按顺序fallback到旧版`masks/{merged,sam,mit}/`（兼容旧布局用户）。没有蒙版文件时直接忽略，不制作也完全没问题。

### 8.3 常用参数（以LoRA默认值为例）

| 参数 | 默认值 | 说明 |
|---|---|---|
| `network_dim` | `32` | LoRA rank。越大表达能力↑，参数量↑ |
| `network_alpha` | `32` | LoRA缩放（通常与`network_dim`相同） |
| `learning_rate` | `2e-5` | 学习率。Hydra可适当降低 |
| `max_train_epochs` | `4` | 数据集越小，epoch数↑ |
| `save_every_n_epochs` | `2`（gui-methods）/ `4`（methods） | 适配器权重累积保存周期 |
| `checkpointing_epochs` | `2`（gui-methods）/ `4`（methods） | 恢复用状态保存周期（单文件覆盖） |
| `caption_dropout_rate` | `0.1` | 将部分描述词替换为空字符串（有助于CFG） |
| `use_shuffled_caption_variants` | `true` | 使用描述词逗号打乱变体 |

各变体开关（`use_ortho`、`use_timestep_mask`、`add_reft`、`use_moe_style`、`router_source`等）通过解除`configs/methods/lora.toml`中注释块来激活，或直接使用`configs/gui-methods/<variant>.toml`中按变体独立的文件。**推荐起点`tlora`变体已预先开启`use_ortho = true` + `use_timestep_mask = true`，即OrthoLoRA + T-LoRA组合。**

### 8.4 训练过程中发生的事情

1. 加载文本编码器 → 生成/验证缓存 → 卸载
2. 加载VAE → 生成/验证缓存 → 卸载
3. *延迟加载*DiT以避免缓存阶段的VRAM冲突
4. 将适配器网络挂载到DiT的attention / FFN模块（各变体目标不同）
5. 采样噪声 → DiT前向传播 → flow-matching损失 → 反向传播 → optimizer step
6. （可选）使用`validation_split`计算验证损失并生成样本图像

### 8.5 输出物

- 训练权重：`output/ckpt/<output_name>.safetensors`（各变体自动区分为`anima`、`anima_tlora_ortho`、`anima_tlora_reft`、`anima_hydra`、`anima_postfix`等）
- 检查点：按`save_every_n_epochs`周期保存到`output/ckpt/`（`.snapshot.toml`附件 + Hydra的`_moe`附带文件）
- 验证样本：`output/ckpt/sample/`
- 推理结果图像：`output/tests/`

### 8.6 自动恢复（checkpointing_epochs）

训练中途中断后，**系统会自动从最后保存的断点继续训练**。对于停电、OOM、误关窗口、暂时关机等情况非常有用。默认已在方法文件中开启，无需额外配置。

```toml
checkpointing_epochs = 2     # 每2个epoch保存一次恢复用状态（覆盖）
```

与`save_every_n_epochs`的区别：

| 参数 | 保存内容 | 是否累积？ | 用途 |
|---|---|---|---|
| `save_every_n_epochs` | 适配器权重（如`anima_lora-000004.safetensors`） | **累积**（或通过`save_last_n_epochs`限制） | 查看中间结果或对比过拟合时间点 |
| `checkpointing_epochs` | 训练恢复完整状态（optimizer / scheduler / RNG / 适配器权重） | **覆盖单一文件** | 训练中断时自动续训 |

工作方式：

- **自动保存**：按`checkpointing_epochs`周期，`output/ckpt/<output_name>-checkpoint-state/`（状态目录）+ `<output_name>-checkpoint.safetensors`（权重）一起保存，覆盖旧版本。磁盘占用不会膨胀。
- **自动恢复**：再次运行相同命令（`make lora`等），若存在已保存的检查点且尚未到达`max_train_steps`，**系统会自动继续训练**。无需手动指定`--resume`标志。日志中出现`auto-resuming from checkpoint at step N`即为正常恢复。
- **自动清理**：训练正常结束后，上述两个文件自动删除——恢复用状态是临时文件，最终成果为`output/ckpt/<output_name>.safetensors`（参见8.5）。
- **手动恢复**：如需恢复到其他时间点的状态，可手动指定`--resume <state_dir>`。

> **何时禁用？** 短期实验训练或磁盘空间不足时，注释掉`checkpointing_epochs`即可。数据集较大的正式训练几乎建议始终开启。
>
> **注意**：更改数据集、描述词或核心配置（rank/LR/epoch数等）后，基于旧检查点继续训练是无意义甚至危险的。此时请手动删除`output/ckpt/<output_name>-checkpoint-state/`目录后重新开始。

## 9. LoRA / 适配器变体选择指南

> **🌟 推荐**：初次使用或制作普通角色/风格LoRA，请从**`tlora`（OrthoLoRA + T-LoRA）**开始。细节/风格均衡和训练稳定性最佳。

| 变体 | 运行方式 | 适用场景 |
|---|---|---|
| **OrthoLoRA + T-LoRA** ⭐ | `make lora-gui GUI_PRESETS=tlora` | **推荐**。SVD正交旋转（OrthoLoRA）+ 时间步rank掩码（T-LoRA）叠加。输出`anima_tlora_ortho.safetensors` |
| **OrthoLoRA + T-LoRA（8GB）** | `make lora-gui GUI_PRESETS=tlora-8gb` 或 `PRESET=low_vram make lora-gui GUI_PRESETS=tlora` | VRAM 8~12GB环境下使用上述推荐组合 |
| **基础LoRA** | `make lora-gui GUI_PRESETS=lora` 或 `make lora` | 最简单的基线，用于对比实验 |
| **基础LoRA（8GB）** | `make lora-gui GUI_PRESETS=lora-8gb` 或 `PRESET=low_vram make lora` | VRAM 8~12GB |
| **T-LoRA + Ortho + ReFT** | `make lora-gui GUI_PRESETS=tlora_ortho_reft` | 在推荐组合基础上增加表示编辑（ReFT），用更少的额外参数进行精调 |
| **HydraLoRA** | `make lora-gui GUI_PRESETS=hydralora`（8GB为`hydralora-8gb`） | MoE多头路由，在单个适配器中处理多种概念 |
| **单独ReFT** | `make lora-gui GUI_PRESETS=reft` 或在`methods/lora.toml`中设置`add_reft = true` | 表示精调（Representation Fine-Tuning），参数量极少 |
| **Postfix Tuning** *（实验性）* | `make exp-postfix` 或 `make lora-gui GUI_PRESETS=postfix_ortho_cond` | cross-attention末尾附加N个可训练向量（描述词条件 + 正交变换） |
| **ChimeraHydra** *（实验性）* | `make exp-chimera` 或 `make lora-gui GUI_PRESETS=chimera_hydra` | 内容/频率双池MoE — 研究用途 |

各变体的详细选项参见[`docs/guidelines/training.md`](training.md)及`docs/methods/`中的各自文档。

> **兼容性说明**
> - HydraLoRA / ReFT等适配器变体需要`cache_llm_adapter_outputs = true`默认开启才能正常工作。
> - `tlora`和`tlora_ortho_reft`的OrthoLoRA + T-LoRA部分可通过`make merge`烘焙到基础DiT中，生成ComfyUI独立检查点（ReFT部分无法烘焙 — 需要`--allow-partial`）。

## 10. 推理

### 10.1 最快速的测试

如需用刚训练好的适配器立即生成样本，请根据变体使用对应的`make test-*`。所有命令都会自动从`output/ckpt/`选取最近保存的对应适配器进行推理。

```bash
make test                        # 普通LoRA / OrthoLoRA / T-LoRA / ReFT
make test SPECTRUM=1             # Spectrum加速推理
make test MOD=1                  # Modulation guidance (pooled_text_proj) — 可与SPECTRUM=1组合
make test NOLORA=1               # 基础DiT单独推理（省略--lora_weight）；与MOD=1组合则为mod-only路径
make test-hydra                  # HydraLoRA (router-live, anima_hydra*_moe.safetensors)
make test-merge                  # 用烘焙的独立DiT（*_merged.safetensors）推理
make test-dcw                    # LoRA + DCW标量校正（采样器步级SNR-t校正）
make test-dcw-v4                 # LoRA + DCW v4可学习校正器
# 实验性推理
make exp-test-postfix            # Postfix tuning（vanilla）
make exp-test-postfix-exp        # postfix_exp变体
make exp-test-postfix-func       # postfix_func变体
```

### 10.2 手动推理

```bash
python inference.py \
    --dit models/diffusion_models/anima-base-v1.0.safetensors \
    --text_encoder models/text_encoders/qwen_3_06b_base.safetensors \
    --vae models/vae/qwen_image_vae.safetensors \
    --lora_weight output/ckpt/anima_lora.safetensors \
    --lora_multiplier 1.0 \
    --prompt "masterpiece, best quality, an anime girl in a sunlit forest" \
    --negative_prompt "worst quality, low quality, blurry" \
    --image_size 1024 1024 \
    --infer_steps 30 \
    --guidance_scale 4.0 \
    --sampler er_sde \
    --flow_shift 1.0 \
    --seed 42 \
    --save_path output/tests
```

常用标志：

| 标志 | 说明 |
|---|---|
| `--lora_weight` | 训练好的适配器路径。可同时指定多个 |
| `--lora_multiplier` | 适配器强度（0.0~1.5） |
| `--image_size H W` | 输出分辨率（如`1024 1024`、`1024 1536`） |
| `--infer_steps` | 去噪步数（通常20~50） |
| `--guidance_scale` | CFG强度（推荐3.0~5.0） |
| `--sampler` | `er_sde`、`euler`、`dpm++`等 |
| `--seed` | 复现用随机种子 |
| `--spectrum` | 开启Spectrum加速 |
| `--pgraft` | P-GRAFT（去噪后期LoRA截止，由基础模型负责后期细节） |

推理选项完整列表及P-GRAFT推理方法参见[`docs/guidelines/inference.md`](inference.md)。

## 11. 部署到ComfyUI

ComfyUI核心原生支持Anima基础DiT（`UNETLoader` / `CLIPLoader`直接加载）。适配器类型不同，部署方式也不同。

### 11.1 经典LoRA / OrthoLoRA / T-LoRA

将`output/ckpt/`中生成的`.safetensors`文件直接复制到`ComfyUI/models/loras/`，即可通过ComfyUI默认LoraLoader节点使用。如需生成更整洁的独立检查点：

```bash
make merge ADAPTER_DIR=output/ckpt                 # 将最新权重烘焙到基础DiT
make merge ADAPTER_DIR=output/ckpt MULTIPLIER=0.8  # 调整强度
```

烘焙后的`*_merged.safetensors`可直接在ComfyUI的`UNETLoader`中作为独立模型加载。

### 11.2 HydraLoRA / ReFT / Postfix

这些变体无法用ComfyUI默认LoraLoader加载（不是简单的权重差值，包含路由/token插入等操作），需要专用节点：

- **Anima Adapter Loader**（`custom_nodes/comfyui-hydralora/`）— 统一处理LoRA / Hydra / ReFT / postfix。详细使用方法参见该目录下的`README.md`。
- **Spectrum KSampler / Mod Guidance / DCW节点** — 独立仓库 <https://github.com/sorryhyun/ComfyUI-Spectrum-KSampler>
