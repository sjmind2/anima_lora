## 2. CUDA 13.2 のインストール

PyTorch 2.x + Flash Attention 2 が安定して動作する最新の CUDA が必要です。NVIDIA 公式アーカイブから 13.2 をダウンロードしてインストールします。

ダウンロードページ: <https://developer.nvidia.com/cuda-13-2-0-download-archive>

### 2.1 Windows へのインストール

1. 上記ページで **Operating System: Windows → Architecture: x86_64 → Version: 11/10 → Installer Type: exe (local)** を選択。
2. ダウンロードした `cuda_13.2_windows.exe` を実行 → 「Express (推奨)」でインストール。
3. インストール後、PowerShell で確認:

   ```powershell
   nvidia-smi
   nvcc --version
   ```

4. `nvcc` が認識されない場合は、システム環境変数 `Path` に以下を追加:

   ```
   C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v13.2\bin
   C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v13.2\libnvvp
   ```

5. 再起動後に再度 `nvcc --version` で確認。

> **ドライバーの注意**: CUDA 13.x には NVIDIA ドライバー 595 以上が必要です。古いドライバーをお使いの場合は、GeForce Experience または NVIDIA ダウンロードセンターで先にアップデートしてください。

## 3. Python 環境とリポジトリの準備

このプロジェクトは依存関係の管理に [`uv`](https://github.com/astral-sh/uv) を使用します。Python 3.13 を使います。

### 3.0 ワンライン インストール (最も簡単、git 不要) ✅

初心者にはこの方法をおすすめします。PowerShell に次の一行を貼り付けると — `uv` がなければ自動インストールし、最新リリースをダウンロードして `uv sync` まで実行し、**デスクトップに「Anima LoRA GUI」のショートカット**を作成します:

```powershell
irm https://raw.githubusercontent.com/sorryhyun/anima_lora/main/install.ps1 | iex
```

`.\anima_lora\` フォルダにインストールされます (`$env:ANIMA_DIR` でパス変更、`$env:ANIMA_VERSION='v1.4.0'` で特定バージョンに固定可能)。完了したら:

1. `cd anima_lora`
2. **§4** で Hugging Face トークンを作成し `hf auth login` でログイン
3. `python tasks.py download-models` でモデルをダウンロード
4. デスクトップの **「Anima LoRA GUI」** ショートカットをダブルクリックして起動 (または `python tasks.py gui`)

> この方法は GUI 中心の利用を前提としており、`make` はインストールされません。ほとんどの作業は GUI ボタンで完結するため問題ありませんが、CLI で `make ...` を使いたい場合は §3.3 の `winget install ezwinports.make` を実行するか、`make` の代わりに `python tasks.py` を使ってください。
>
> 以降のアップデートはフォルダ内で `python tasks.py update` (リリース tarball マージ、git 不要)。

下記の **§3.1–§3.3** は、`git clone` で直接取得したい方や各ステップを理解したい方向けの手動インストール手順です。

### 3.1 `uv` のインストール

```powershell
irm https://astral.sh/uv/install.ps1 | iex
```

インストール後、新しいシェルを開いて `uv --version` が表示されることを確認してください。

### 3.2 リポジトリのクローン

```bash
git clone https://github.com/sorryhyun/anima_lora.git
cd anima_lora
```

> このガイドではすべてのパスを `anima_lora/` ディレクトリ基準で説明します。すべてのコマンドはこのフォルダ内で実行してください。

### 3.3 依存関係のインストール

```bash
winget install ezwinports.make
uv sync
```

`uv sync` は `pyproject.toml`/`uv.lock` を参照して、**`anima_lora/` 内に `.venv/` フォルダ(独立した Python 環境)を作成し**、すべての依存関係をその中にインストールします。システムの Python には手を加えないため、他のプロジェクトに影響しません — これが正常な動作で、自分で `pip install` する必要はありません。

> **初心者が最もつまずきやすいポイントです。** `uv sync` が終わると、パッケージはグローバルな Python ではなく `.venv/` の中に入っています。ターミナルを開いていきなり `python tasks.py ...` や `make lora` を実行すると、そのシェルはまだ*システム*の Python を使っているため依存関係が無く、`ModuleNotFoundError` になりがちです。コマンドが `.venv/` の Python を指すようにする必要があり、方法は 2 つあります。

**方法 A — `uv run` (有効化不要、どこでも動く)。** どんなコマンドにも `uv run` を前置すると、自動的に `.venv/` を使います:

```bash
uv run make lora              # または: uv run python tasks.py lora
uv run hf auth login
uv run make download-models
```

シェルの状態に依存しないため環境を「忘れる」ことがなく、最も確実な方法です。

**方法 B — ターミナルごとに一度 venv を有効化。** 有効化すると、そのシェルセッションの間 `.venv/` の `python` が `PATH` の最優先になり、`uv run` を付けずに `make ...` / `python tasks.py ...` がそのまま動きます:

```powershell
.venv\Scripts\activate        # Windows (PowerShell / cmd)
```
```bash
source .venv/bin/activate     # Linux / macOS / WSL
```

プロンプトに `(anima_lora)` (または `(.venv)`) という接頭辞が表示されれば成功です。**新しいターミナルウィンドウごとに再度有効化が必要です** — 有効化状態は持続しません。終了するには `deactivate`。VSCode ユーザーは `.venv` インタープリタを一度選択しておけば(コマンドパレット → *Python: Select Interpreter*)、統合ターミナルが自動で有効化されます。

> このガイドの残りのコマンドは普通の `make ...` / `python tasks.py ...` の形で書かれており、venv を有効化した(方法 B)か `uv run` を前置する(方法 A)ことを前提としています。コマンドが `ModuleNotFoundError` や `command not found: make` で失敗する場合、ほぼ必ずこれが原因です — venv を有効化するか `uv run` を付けてください。

## 5. データセットの準備

Anima LoRA は *画像 + 同名の `.txt` キャプションサイドカー* の構成を使います。以下は `image_dataset/` フォルダの例です。

```
image_dataset/
├─ 00001.png
├─ 00001.txt
├─ 00002.jpg
├─ 00002.txt
├─ subfolder/
│  ├─ 00010.webp
│  └─ 00010.txt
└─ ...
```

### 5.1 キャプション作成のヒント

- Anima 公式ガイドラインに従うと、タグの順序は常に [meta] [character] [series] [artist] [general] です。例えば以下のように書きます:

```
absurdres, safe, 1girl, chitanda eru, hyouka, @channel (caststation), full body, serafuku, She is saying hi.
```

- この際、absurdres・highres・masterpiece などのクオリティタグは、個人的な実験の結果、書かないかできるだけ最小限にとどめるのが最善です。あるいは今後正式配布される mod guidance を使えば一切書かなくても大丈夫です。
- 元画像は `image_dataset/` に置きます (ファイル名は自由、この場所をそのまま使ってください)。

### 5.2 `num_repeats` とは何か、いつ触るのか? (要約: **触らない**)

`configs/base.toml` の `[[datasets.subsets]]` の中に `num_repeats = 1` という項目が見えるはずです。この値は **1 エポック中に各画像を何回繰り返し使うか** を指定する kohya-ss スタイルのオプションで、他の LoRA トレーナーのガイドを参考にしていると頻繁に目にするキーです。

- **このガイドブックの標準ワークフローでは `1` のままにしてください。** すべての画像を `image_dataset/` 一つのフォルダにまとめて学習する一般的な使い方では、`num_repeats` を上げても *1 エポックが長くなるだけ* で、効果は `max_train_epochs` を増やすのと同じです。学習量は常にエポック数で調節するのが直感的で、本プロジェクトのすべてのプリセット・メソッド設定は `num_repeats = 1` を前提にチューニングされています。
- **いつ上げると意味があるのか?** 一つの学習に *異なるサブセット (フォルダ) が複数* あり、フォルダごとの画像数が大きく異なるとき、小さいフォルダの露出頻度を大きいフォルダに合わせるための *バランシングツール* としてのみ意味があります (例: キャラクター A が 1000 枚・キャラクター B が 50 枚なら B のサブセットだけ `num_repeats = 20`)。単一フォルダでの学習には該当しません。
- **どこで変更するのか?** `num_repeats` はメソッド設定ではなく *データセット設定* なので、`configs/methods/`・`configs/gui-methods/`・GUI の学習タブには表示されません。本当に必要な場合は `configs/base.toml` の `[[datasets.subsets]]` (または `--dataset_config <path>` で別途指定した TOML) を直接編集してください。*同じ画像を単純にもっと多く学習させたい場合*は `num_repeats` ではなく `max_train_epochs` を増やすのが正解です。

## 6. 前処理: リサイズ・潜在変数・テキスト埋め込みのキャッシング

学習速度と VRAM を最適化するために、**リサイズ → VAE 潜在変数キャッシング → テキスト埋め込みキャッシング** の 3 ステップを事前に実行します。

```bash
make preprocess              # 3 ステップすべてを実行 (LoRA / 一般学習用)
# またはステップ別に
make preprocess-resize       # 1) image_dataset/ → post_image_dataset/resized/
make preprocess-vae          # 2) VAE 潜在変数キャッシング → post_image_dataset/lora/
make preprocess-te           # 3) テキストエンコーダー出力キャッシング → post_image_dataset/lora/
make preprocess-pe           # (オプション) PE-Core ビジョンエンコーダー特徴量キャッシング — IP-Adapter 専用
```

> **⚠️ キャッシュは再利用されます — 自動削除は絶対にされません。**
> `make preprocess` (および GUI の *Preprocess* ボタン) は **すでに作成済みのキャッシュをそのまま再利用します**。`post_image_dataset/lora/` 内の `.npz` / `_te.safetensors` / `_pe.safetensors` ファイルは *上書きも削除もされず*、欠落しているエントリのみ新たに処理されます。そのため再実行が非常に高速で、途中で中断しても安全です。
>
> つまり、すでにキャッシュがある状態で `make preprocess` を再実行しても既存データは失われないのでご安心ください。逆に **キャプション / トークナイザー / リサイズオプションを変更して *最初から* 作り直したい場合**は、キャッシュフォルダ (`post_image_dataset/lora/` または `post_image_dataset/easycontrol/`) を直接削除してから再実行する必要があります。

### 6.1 リサイズが行うこと

- VAE が要求するピクセル配置に合わせて画像をリサイズ
- *(H/16) × (W/16) = 4032 または 4200 パッチ*という 2 つのトークン数系列の *固定トークンバケット* に自動分類 (各バケットはそのトークン数をちょうど満たす)
- 小さすぎる画像 (デフォルト 0.5MP 未満) は自動除外して報告
- 結果は `post_image_dataset/resized/` に PNG として保存

### 6.2 潜在変数のキャッシング

- すべてのリサイズ画像に対して VAE を 1 回だけ実行して結果をディスクに保存
- 学習中は VAE を GPU に乗せないため VRAM が大幅に節約される
- キャッシュ保存先: `post_image_dataset/lora/{stem}_{WxH}_anima.npz`
- スクリプト: `scripts/preprocess/cache_latents.py`

### 6.3 テキスト埋め込みのキャッシング

- Qwen3 0.6B + LLM アダプター出力を事前計算
- `use_shuffled_caption_variants = true` の場合はキャプションのコンマシャッフルバリアントも一緒にキャッシング (学習時にランダム選択)
- キャッシュ保存先: `post_image_dataset/lora/{stem}_anima_te.safetensors`
- キャプションは常に `image_dataset/` のオリジナル `.txt` を読み込みます (リサイズフォルダにはコピーしない)
- スクリプト: `scripts/preprocess/cache_text_embeddings.py`

### 6.4 PE ビジョン特徴量のキャッシング (オプション)

- IP-Adapter 学習 (および DCW v4 fusion head) に必要
- PE-Core-L14-336 ビジョンエンコーダーの出力を事前計算して、学習中にビジョンエンコーダーを乗せないようにする
- キャッシュ保存先: `post_image_dataset/lora/{stem}_anima_pe.safetensors`

> **キャッシュをいつ作り直すのか?**
> - 画像が追加されたとき → `make preprocess` を再実行するだけです (既存キャッシュは維持され、新しいエントリのみ追加されます)。
> - **キャプションを編集したとき**や**トークナイザー/パディング関連オプションを変更したとき** → 上記の ⚠️ に従いキャッシュフォルダ (`post_image_dataset/lora/`) を直接削除してから再実行してください。単純な再実行は *既存キャッシュをそのまま再利用するため* 変更が反映されません。

## 7. GUI の使い方

PySide6 ベースの GUI を使うと、設定の編集・データセットのブラウズ・前処理・学習の実行/モニタリング・LoRA のマージを一画面で行えます。

```bash
make gui
make gui-shortcut   # (オプション) Windows デスクトップにコンソールウィンドウなしの .lnk ショートカットを作成
```

GUI の主要タブ:

- **学習設定 (Training Config)**: LoRA 系バリアント (推奨: `tlora` — Ortho + T-LoRA / その他 `lora`・`tlora-8gb`・`tlora_ortho_reft`・`hydralora`・`reft` など) をドロップダウンで選択し、`presets.toml` プリセット (default / low_vram など) とすべての学習キーを直接編集してそのまま学習開始
- **前処理 (Preprocess)**: リサイズ + VAE + テキスト埋め込みキャッシングを一度に
- **データセット (Dataset)**: 画像/キャプションのプレビューとキャプションの直接編集
- **マージ (Merge)**: 学習した LoRA をベース DiT にベイクして ComfyUI 単体チェックポイントとして保存 (基本の LoRA / OrthoLoRA / T-LoRA のみ対応)

GUI での学習も内部的には `train.py` を呼び出すため、同じパラメーターを CLI でも再現できます。GUI は `configs/gui-methods/<variant>.toml` (トグルブロックなしのバリアント別単一ファイル) を読み込むため、GUI に表示されるバリアント一覧と CLI の `make lora-gui GUI_PRESETS=<variant>` が同じ結果を返します。バリアント一覧の現在の状態は `ls configs/gui-methods/` で確認してください。

### 7.1 フォームの編集と保存の動作

学習/前処理のサブプロセスはディスク上のバリアント TOML ファイルを再読み込みするため、フォームを編集しただけで保存しないと変更が学習に反映されません。GUI はこれを 2 つの方法で処理します。

- **変更の検知**: フォームのいずれかのフィールド (または `+ Extra args` テキストボックス) を編集すると、`Save` ボタンがオレンジ色になり `Save *` の表示が付きます。つまり *ディスクのバリアントファイルと画面が異なる* というシグナルです。`Save` を押すか、バリアントを再選択してディスクから再読み込みすると表示が消えます。
- **自動保存**: それでも保存し忘れて `Train` / `Preprocess` を押した場合、その時点のフォームの値が自動的にバリアントファイルに先に書き込まれてからサブプロセスが実行されます。つまり画面に表示されている値がそのまま学習に使われる値です (`Test` は最後の学習結果のチェックポイントで推論するため自動保存の対象外です)。

> 変更を *試すだけ* にしたい場合は、フォームを編集した後 `Train` を押さずに別のバリアントに切り替えてから戻ると、ディスクから再読み込みされて編集が破棄されます。

### 7.2 自動再開 (checkpointing_epochs)

学習が途中で中断されても、`Train` を再度押すと **自動的に最後に保存したポイントから学習が継続**されます。停電・OOM・誤ってウィンドウを閉じた場合などへの備えとして最も便利な機能の一つで、デフォルトで有効になっています。

GUI での扱い方:

- 学習設定タブの **Training グループ**に `checkpointing_epochs` フィールドがあります (gui-methods バリアントのデフォルトは `2`、`methods/lora.toml` のデフォルトは `4`)。N エポックごとに再開用の状態を保存し、同じファイルを上書きするためディスクが肥大化しません。
- 学習が中断された後、同じバリアントで `Train` を再度押すと、ログウィンドウに `auto-resuming from checkpoint at step N` と表示され、そのポイントから学習が継続されます。別途フラグを設定する必要はありません。
- 学習が正常に完了すると、再開用の一時ファイルは自動削除され、最終結果は `output/ckpt/<output_name>.safetensors` として残ります。
- **データセットやコア設定 (ランク・LR・エポック数など) を変更した後**に新規学習する場合は、`output/ckpt/<output_name>-checkpoint-state/` フォルダを手動で削除してから `Train` を押してください。そうしないと古い状態の上から学習が継続されます。

詳細な動作は [§8.6 自動再開](#86-自動再開-checkpointing_epochs) で説明します — `save_every_n_epochs` との違いもそこで整理しています。

### 7.3 学習の停止と GUI のクローズ

学習は GUI ウィンドウの*中*で動いているわけではありません — `Train` を押すと、ジョブは `train.py` を分離(detached)プロセスとして実行する小さなバックグラウンド **学習デーモン** に引き渡されます。これには実用上 2 つの帰結があります:

- **`Stop` ボタンは現在の学習ジョブを中断します。** デーモンは動き続け、(キューがあれば) 次のジョブへ進むため、1 つのジョブを止めてもキュー全体が壊れることはありません。CLI の `make daemon-kill` と同じです。(`Stop` は進行中の `Test` や `Preprocess` もキャンセルします。この 2 つは GUI の*中*で動きます。)
- **GUI を閉じても学習は止まりません。** ジョブは分離デーモンで動くため、ウィンドウを閉じても学習は続きます — 一晩中回しておきたい長時間の学習に便利です。GUI を再度開くと(`make gui`)、まだ動いているジョブに自動的に再接続し、ログに `Re-attached to running job …` が表示され、プログレスバーと出力がリアルタイムで再開します。(CLI や ComfyUI トレーナーノードから開始したジョブもこうして現れます。)

> 学習を完全に終了するには — 進行中のジョブを終了し*かつ*デーモンも止めて GPU を解放するには — CLI で `make daemon-terminate` を使ってください。`Stop` だけではデーモンは起動したままです。
>
> `Test` と `Preprocess` は例外です: ウィンドウ内のサブプロセスとして動くため、GUI を閉じるとキャンセルされます。

## 8. 学習の実行

すべての学習は TOML 設定ファイルと HuggingFace Accelerate を通じて実行されます。設定のマージ順序は `configs/base.toml → configs/presets.toml[<preset>] → configs/methods/<method>.toml → CLI args` で、メソッド設定がプリセットを上書きします。

### 8.1 クイックスタート

**最も推奨される出発点は OrthoLoRA + T-LoRA (= `tlora` バリアント)** です。安定性・ディテール・スタイル保持のバランスが最も取れた組み合わせで、一般的なキャラクター/スタイル LoRA の学習にそのまま使えます。

```bash
# 推奨: Ortho + T-LoRA (gui-methods/tlora.toml)
make lora-gui GUI_PRESETS=tlora                       # 一般環境
PRESET=low_vram make lora-gui GUI_PRESETS=tlora-8gb   # VRAM 8〜12GB

# その他のバリアント (configs/gui-methods/<variant>.toml — トグルブロックなしの単一ファイル)
make lora-gui GUI_PRESETS=lora                        # シンプルな LoRA のみ
make lora-gui GUI_PRESETS=tlora_ortho_reft            # Ortho + T-LoRA + ReFT スタック
make lora-gui GUI_PRESETS=hydralora                   # MoE マルチヘッド
make lora-gui GUI_PRESETS=reft                        # ReFT 単体

# トグルブロック方式 (configs/methods/lora.toml 内で直接バリアントを選択)
make lora                          # presets.toml[default]
PRESET=low_vram make lora          # presets.toml[low_vram] — VRAM 8〜12GB
PRESET=half make lora              # データセットの半分のみ使って素早く実験
```

> **CLI から直接キーをオーバーライド**: `make lora -- --network_dim 32 --max_train_epochs 24` のように追加引数をそのまま渡せます (`tasks.py` も同様)。

### 8.2 マスクド損失 (テキストバブルの除外)

漫画・アニメ風データでセリフバブルやテキスト領域を学習損失から除外すると、結果が大幅にクリーンになります。

```bash
make mask          # SAM3 + MIT (一時ディレクトリ経由) → post_image_dataset/masks/
make mask-clean    # post_image_dataset/masks/ を削除
```

生成される PNG は白黒です: **白 (255) = 学習対象**、**黒 (0) = 除外**。データセットサブセットは `post_image_dataset/masks/` があればそれを自動的に使用し、なければレガシーの `masks/{merged,sam,mit}/` を順番にフォールバックします (以前のレイアウトをそのまま使っているユーザーでも動作します)。マスクファイルがなければ単に無視されるため、作成しなくても問題ありません。

### 8.3 よく変更する設定値 (LoRA 基準のデフォルト値)

| パラメーター | デフォルト値 | 説明 |
|---|---|---|
| `network_dim` | `32` | LoRA ランク。大きいほど表現力↑、パラメーター数↑ |
| `network_alpha` | `32` | LoRA スケール (通常 `network_dim` と同じ値) |
| `learning_rate` | `2e-5` | 学習率。Hydra ではさらに低くしても OK |
| `max_train_epochs` | `4` | データセットが小さいほどエポック数↑ |
| `save_every_n_epochs` | `2` (gui-methods) / `4` (methods) | アダプター重みの累積保存間隔 |
| `checkpointing_epochs` | `2` (gui-methods) / `4` (methods) | 再開用状態の保存間隔 (単一ファイルを上書き) |
| `caption_dropout_rate` | `0.1` | キャプションの一部を空文字列に置換 (CFG に有効) |
| `use_shuffled_caption_variants` | `true` | キャプションのコンマシャッフルバリアントを使用 |

各バリアントのトグル (`use_ortho`・`use_timestep_mask`・`add_reft`・`use_moe_style`・`router_source` など) は `configs/methods/lora.toml` のコメントブロックを解除して有効化するか、`configs/gui-methods/<variant>.toml` でバリアント別の単一ファイルをそのまま使ってください。**推奨の出発点である `tlora` バリアントは `use_ortho = true` + `use_timestep_mask = true` が最初から有効になっている OrthoLoRA + T-LoRA の組み合わせです。**

### 8.4 学習中に行われること

1. テキストエンコーダーのロード → キャッシュの生成/検証 → アンロード
2. VAE のロード → キャッシュの生成/検証 → アンロード
3. DiT を *遅延読み込み* してキャッシング段階の VRAM 競合を防止
4. アダプターネットワークを DiT の attention / FFN モジュールにパッチ (バリアントごとに対象が異なる)
5. ノイズサンプリング → DiT フォワード → flow-matching 損失 → バックワード → オプティマイザーステップ
6. (オプション) `validation_split` で検証損失の測定とサンプル画像の生成

### 8.5 出力物

- 学習済み重み: `output/ckpt/<output_name>.safetensors` (バリアントごとに `anima`・`anima_tlora_ortho`・`anima_tlora_reft`・`anima_hydra`・`anima_postfix` などに自動分岐)
- チェックポイント: `save_every_n_epochs` 間隔ごとに `output/ckpt/` に保存 (`.snapshot.toml` サイドカー + Hydra の場合は `_moe` 付随ファイル)
- 検証サンプル: `output/ckpt/sample/`
- 推論結果画像: `output/tests/`

### 8.6 自動再開 (checkpointing_epochs)

学習が途中で中断されても **最後に保存したポイントから自動的に再開** できる機能です。停電・OOM・誤ってウィンドウを閉じた・少し PC を切って休みたいときなどに非常に便利です。デフォルトのメソッドファイルですでに有効化されているため、別途設定は不要です。

```toml
checkpointing_epochs = 2     # 2 エポックごとに再開用状態を保存 (上書き)
```

`save_every_n_epochs` とは役割が異なります。

| キー | 保存対象 | 蓄積? | 用途 |
|---|---|---|---|
| `save_every_n_epochs` | アダプター重み (`anima_lora-000004.safetensors` など) | **蓄積** (または `save_last_n_epochs` で制限) | 途中経過を推論して過学習のタイミングを比較 |
| `checkpointing_epochs` | 学習再開用フル状態 (オプティマイザー / スケジューラー / RNG / アダプター重み) | **単一ファイルを上書き** | 学習中断時の自動再開 |

動作の仕組み:

- **自動保存**: `checkpointing_epochs` 間隔で `output/ckpt/<output_name>-checkpoint-state/` (状態ディレクトリ) + `<output_name>-checkpoint.safetensors` (重み) が一緒に保存され、前のものを上書きします。ディスクが肥大化しません。
- **自動再開**: 同じコマンド (`make lora` など) を再度実行すると、保存済みチェックポイントがあり `max_train_steps` に達していなければ **自動的に学習が継続されます**。`--resume` のようなフラグを直接指定する必要はありません。ログに `auto-resuming from checkpoint at step N` が表示されれば正常に再開されています。
- **自動クリーンアップ**: 学習が正常に終了すると上記の 2 ファイルは自動削除されます — 再開用状態は一時ファイルであり、最終成果物は `output/ckpt/<output_name>.safetensors` です (8.5 参照)。
- **手動再開**: 別のポイントの状態に戻したい場合は `--resume <state_dir>` を直接指定することもできます。

> **無効にする場合**: 短い実験学習やディスクが不足している場合は `checkpointing_epochs` をコメントアウトしてください。データセットが大きい本番学習ではほぼ常に有効にしておくことを推奨します。
>
> **注意**: データセット・キャプション・学習設定 (ランク・LR・エポック数など) を変更した場合、古いチェックポイントから再開することは意味がないか危険です。この場合は `output/ckpt/<output_name>-checkpoint-state/` フォルダを手動で削除してから新規に開始してください。

## 9. LoRA / アダプターバリアント選択ガイド

> **🌟 推奨**: 初めての方や一般的なキャラクター・スタイル LoRA を作る場合は **`tlora` (OrthoLoRA + T-LoRA)** から始めてください。ディテール/スタイルのバランスと学習安定性が最も優れています。

| バリアント | 実行方法 | どんなときに使うか |
|---|---|---|
| **OrthoLoRA + T-LoRA** ⭐ | `make lora-gui GUI_PRESETS=tlora` | **推奨**。SVD ベースの直交回転 (OrthoLoRA) + タイムステップ別ランクマスキング (T-LoRA) スタック。`anima_tlora_ortho.safetensors` を出力 |
| **OrthoLoRA + T-LoRA (8GB)** | `make lora-gui GUI_PRESETS=tlora-8gb` または `PRESET=low_vram make lora-gui GUI_PRESETS=tlora` | VRAM 8〜12GB 環境で上記の推奨組み合わせをそのまま使用 |
| **シンプルな LoRA** | `make lora-gui GUI_PRESETS=lora` または `make lora` | 最もシンプルなベースライン、比較実験用 |
| **シンプルな LoRA (8GB)** | `make lora-gui GUI_PRESETS=lora-8gb` または `PRESET=low_vram make lora` | VRAM 8〜12GB |
| **T-LoRA + Ortho + ReFT** | `make lora-gui GUI_PRESETS=tlora_ortho_reft` | 上記推奨組み合わせに表現編集 (ReFT) を加えてより少ない追加パラメーターで微調整 |
| **HydraLoRA** | `make lora-gui GUI_PRESETS=hydralora` (8GB は `hydralora-8gb`) | MoE マルチヘッドルーティング、様々なコンセプトを一つのアダプターに |
| **ReFT 単体** | `make lora-gui GUI_PRESETS=reft` または `methods/lora.toml` で `add_reft = true` | 表現編集 (Representation Fine-Tuning)、非常に少ないパラメーター |
| **Postfix Tuning** *(実験的)* | `make exp-postfix` または `make lora-gui GUI_PRESETS=postfix_ortho_cond` | クロスアテンションの末尾に学習可能な N 個のベクトル (キャプション条件付き + 直交バリアント) |
| **ChimeraHydra** *(実験的)* | `make exp-chimera` または `make lora-gui GUI_PRESETS=chimera_hydra` | コンテンツ/周波数デュアルプール MoE — 研究用 |

各バリアントの詳細なオプションは [`docs/guidelines/training.md`](training.md) および `docs/methods/` の個別ドキュメントを参照してください。

> **互換性メモ**
> - HydraLoRA / ReFT などのアダプターバリアントは `cache_llm_adapter_outputs = true` がデフォルトで有効になっている必要があります。
> - `tlora` と `tlora_ortho_reft` の OrthoLoRA + T-LoRA 部分は `make merge` でベース DiT にベイクして単体の ComfyUI チェックポイントにできます (ReFT 部分はベイク不可 — `--allow-partial` が必要)。

## 10. 推論

### 10.1 最速のテスト

たった今学習したアダプターで即座にサンプルを生成したい場合は、バリアントに合った `make test-*` を使ってください。いずれも `output/ckpt/` から最も最近保存されたアダプターを自動的に選択して推論します。

```bash
make test                        # 一般 LoRA / OrthoLoRA / T-LoRA / ReFT
make test SPECTRUM=1             # Spectrum 加速推論
make test MOD=1                  # Modulation guidance (pooled_text_proj) — SPECTRUM=1 と組み合わせ可能
make test NOLORA=1               # ベース DiT 単体推論 (--lora_weight を省略); MOD=1 と合わせると mod-only パス
make test-hydra                  # HydraLoRA (router-live, anima_hydra*_moe.safetensors)
make test-merge                  # ベイク済み単体 DiT (*_merged.safetensors) で推論
make test-dcw                    # LoRA + DCW スカラー補正 (サンプラーステップ SNR-t 補正)
make test-dcw-v4                 # LoRA + DCW v4 学習型補正器
# 実験的推論
make exp-test-postfix            # Postfix tuning (vanilla)
make exp-test-postfix-exp        # postfix_exp バリアント
make exp-test-postfix-func       # postfix_func バリアント
```

### 10.2 一般推論 (手動)

```bash
python inference.py \
    --dit models/diffusion_models/anima-base-v1.0.safetensors \
    --text_encoder models/text_encoders/qwen_3_06b_base.safetensors \
    --vae models/vae/qwen_image_vae.safetensors \
    --lora_weight output/ckpt/anima_lora.safetensors \
    --lora_multiplier 1.0 \
    --prompt "masterpiece, best quality, an anime girl in a sunlit forest" \
    --negative_prompt "worst quality, low quality, blurry" \
    --image_size 1024 1024 \
    --infer_steps 30 \
    --guidance_scale 4.0 \
    --sampler er_sde \
    --flow_shift 1.0 \
    --seed 42 \
    --save_path output/tests
```

よく使うフラグ:

| フラグ | 説明 |
|---|---|
| `--lora_weight` | 学習済みアダプターのパス。複数同時指定も可能 |
| `--lora_multiplier` | アダプターの強度 (0.0〜1.5) |
| `--image_size H W` | 出力解像度 (例: `1024 1024`、`1024 1536`) |
| `--infer_steps` | デノイジングステップ数 (通常 20〜50) |
| `--guidance_scale` | CFG 強度 (3.0〜5.0 推奨) |
| `--sampler` | `er_sde`・`euler`・`dpm++` など |
| `--seed` | 再現用シード |
| `--spectrum` | Spectrum 加速を有効化 |
| `--pgraft` | P-GRAFT (デノイズ後半で LoRA をカットオフ) でベースモデルが後半のディテールを担当 |

推論オプションの完全なリストと P-GRAFT 推論方法は [`docs/guidelines/inference.md`](inference.md) にあります。

## 11. ComfyUI での配布

ComfyUI コアは Anima ベース DiT をネイティブサポートしています (`UNETLoader` / `CLIPLoader` でそのままロード可能)。アダプターの種類によって配布方法が異なります。

### 11.1 クラシック LoRA / OrthoLoRA / T-LoRA

`output/ckpt/` から出力された `.safetensors` をそのまま `ComfyUI/models/loras/` にコピーすれば、ComfyUI 標準の LoraLoader ノードで直接使用できます。よりクリーンな単体チェックポイントにしたい場合は:

```bash
make merge ADAPTER_DIR=output/ckpt                 # 最新の重みをベース DiT にベイク
make merge ADAPTER_DIR=output/ckpt MULTIPLIER=0.8  # 強度を調整
```

ベイクされた `*_merged.safetensors` は ComfyUI の `UNETLoader` で単体モデルとしてそのままロードできます。

### 11.2 HydraLoRA / ReFT / Postfix

これらのバリアントは ComfyUI 標準の LoraLoader ではロードできません (単純な重みデルタではなく、ルーティング・トークン挿入が含まれるため)。専用ノードが必要です:

- **Anima Adapter Loader** (`custom_nodes/comfyui-hydralora/`) — LoRA / Hydra / ReFT / postfix を統合処理。詳細な使い方は該当フォルダの `README.md` を参照。
- **Spectrum KSampler / Mod Guidance / DCW ノード** — 別リポジトリ <https://github.com/sorryhyun/ComfyUI-Spectrum-KSampler>
