# CAME オプティマイザ — Confidence-guided Adaptive Matrix Evaluation

CAME は、フル行列の二次モーメント（`exp_avg_sq`）を行モーメントと列モーメント（`exp_avg_sq_row`、`exp_avg_sq_col`）で置き換える因子分解型オプティマイザです。2次元以上のパラメータに対するオプティマイザの状態メモリを大幅に削減します。また、因子分解による近似誤差を補正する残余補正機構を備えています。

CAME は[ワークフローエンジン](workflow.ja.md)のデフォルトオプティマイザであり、因子分解モジュールが小規模ながら多数のパラメータ行列を持つ LyCORIS 系トレーニング（LoKR、LoHA）に特に適しています。

## 使用方法

### CLI / TOML 設定

メソッドまたはプリセットの TOML で `optimizer_type` を設定します：

```toml
optimizer_type = "CAME"
learning_rate = 1.5e-5
```

オプティマイザ固有の引数は `optimizer_args` で指定します：

```toml
optimizer_type = "CAME"
optimizer_args = ["weight_decay=0.01", "betas=0.9,0.999,0.9999"]
learning_rate = 1.5e-5
```

### ワークフロー

CAME は Workflow スキーマ（`workflow/schemas/train_common.yaml`）のデフォルトオプティマイザです。設定は不要で、明示的に変更しない限り自動的に選択されます。

## パラメータ

| パラメータ | デフォルト | 説明 |
|-----------|-----------|------|
| `betas` | (0.9, 0.999, 0.9999) | （exp_avg 用 beta0、二次モーメント用 beta1、残余モーメント用 beta2） |
| `eps` | (1e-30, 1e-16) | （更新の安定性用 eps0、残余の安定性用 eps1） |
| `clip_threshold` | 1.0 | RMS クリッピングの閾値 — 更新の爆発を防止します |
| `weight_decay` | 0.0 | 重み減衰係数 |
| `lr` | — | 学習率。**推奨：同等の AdamW 学習率の 0.5–0.9 倍。** |

### 内部動作

- **2D 以上のパラメータ**：行/列の二次モーメント + 残余補正による因子分解更新。同じ形状のパラメータはバッチテンソルにスタックされて効率化されます。
- **1D パラメータ**（バイアス、スカラー）：因子分解なしの Adam スタイルの更新（フル `exp_avg_sq` を使用）。

## 他のオプティマイザとの比較

| オプティマイザ | メモリ（パラメータあたり） | 自動 LR | 最適な用途 | ソース |
|---------------|-------------------------|---------|-----------|--------|
| **AdamW** | 2 × フル状態 | なし | 汎用、デフォルト | `torch.optim` |
| **CAME** | ~2 × (行+列) 状態 | なし | LyCORIS 系、メモリ制約環境 | 組み込み（`library/training/came_optimizer.py`） |
| **Adopt_Adv** | 設定可能（因子分解オプションあり） | なし | 小バッチ安定性、長期トレーニング | `adv_optm` パッケージ |
| **Prodigy_Adv** | 2 × フル状態 + D-Adaptation 状態 | **あり**（`lr=1.0` を設定） | 最適 LR が不明な場合 | `adv_optm` パッケージ |

高度なオプティマイザガイド（Adopt_Adv、Prodigy_Adv）の詳細は [docs/optimizations/adv_optm_guide.md](../optimizations/adv_optm_guide.md) を参照してください。

## 推奨学習率

CAME の因子分解された二次モーメント推定は、AdamW のフル推定とは異なる挙動を示します。実践的な目安：

| シナリオ | AdamW lr | CAME lr |
|----------|----------|---------|
| LoRA rank 32 | 2e-4 | 1–1.5e-4 |
| LoKR dim 8 | 2e-5 | 1.5e-5 |
| LoHA dim 32 | 1e-4 | 7e-5 |

経験則：**AdamW の学習率の 0.7 倍から開始**し、調整してください。

## 既知の制限

- **1D パラメータ**は因子分解なし（Adam スタイル）の更新を使用するため、バイアスやスカラーに対するメモリ削減効果はありません。
- **torch.compile のグループスタック**は、同じ形状のパラメータをバッチテンソルにグループ化します。多数の異なる形状が存在する場合、グループ化のオーバーヘッドがメリットを減少させる可能性があります。ただし、実用上問題になることは稀です。
- **勾累積係数（gradient accumulation factor）> 1 との組み合わせ**には注意が必要です。ステップごとの状態更新は正しいですが、実効的な lr スケーリングが AdamW と異なる場合があります。
