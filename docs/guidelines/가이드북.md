## 2. CUDA 13.2 설치

PyTorch 2.x + Flash Attention 2가 안정적으로 동작하는 최신 CUDA 이어야 합니다. NVIDIA 공식 아카이브에서 13.2를 받아 설치합니다.

다운로드 페이지: <https://developer.nvidia.com/cuda-13-2-0-download-archive>

### 2.1 Windows 설치

1. 위 페이지에서 **Operating System: Windows → Architecture: x86_64 → Version: 11/10 → Installer Type: exe (local)** 선택.
2. 받은 `cuda_13.2_windows.exe` 실행 → "Express (권장)" 설치.
3. 설치 후 PowerShell에서 확인:

   ```powershell
   nvidia-smi
   nvcc --version
   ```

4. `nvcc`가 인식되지 않으면 시스템 환경 변수 `Path`에 다음을 추가:

   ```
   C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v13.2\bin
   C:\Program Files\NVIDIA GPU Computing Toolkit\CUDA\v13.2\libnvvp
   ```

5. 재부팅한 뒤 다시 `nvcc --version`으로 확인.

> **드라이버 주의**: CUDA 13.x는 NVIDIA 드라이버 595 이상이 필요합니다. 구형 드라이버라면 GeForce Experience 또는 NVIDIA 다운로드 센터에서 먼저 업데이트하세요.

## 3. Python 환경과 저장소 준비

이 프로젝트는 [`uv`](https://github.com/astral-sh/uv)로 의존성을 관리합니다. Python 3.13을 사용합니다.

### 3.0 한 줄 설치 (가장 쉬운 방법, git 불필요) ✅

초보자라면 이 방법을 추천합니다. PowerShell에 아래 한 줄을 붙여넣으면 — `uv`가 없으면 자동 설치하고, 최신 릴리스를 받아 `uv sync`까지 끝낸 뒤, **바탕화면에 "Anima LoRA GUI" 바로가기**를 만들어줍니다:

```powershell
irm https://raw.githubusercontent.com/sorryhyun/anima_lora/main/install.ps1 | iex
```

`.\anima_lora\` 폴더에 설치됩니다 (`$env:ANIMA_DIR` 로 경로 변경, `$env:ANIMA_VERSION='v1.4.0'` 으로 특정 버전 고정 가능). 끝나면:

1. `cd anima_lora`
2. **§4** 로 Hugging Face 토큰을 만들고 `hf auth login` 으로 로그인
3. `python tasks.py download-models` 로 모델 다운로드
4. 바탕화면의 **"Anima LoRA GUI"** 바로가기를 더블클릭해 실행 (또는 `python tasks.py gui`)

> 이 방법은 GUI 중심 사용을 전제로 하며 `make` 를 설치하지 않습니다. 대부분의 작업은 GUI 버튼으로 끝나므로 문제 없지만, CLI 로 `make ...` 명령을 쓰고 싶다면 §3.3 의 `winget install ezwinports.make` 를 실행하거나 `make` 대신 `python tasks.py` 를 쓰세요.
>
> 이후 업데이트는 폴더 안에서 `python tasks.py update` (릴리스 tarball 병합, git 불필요).

아래 **§3.1–§3.3** 은 `git clone` 으로 직접 받고 싶거나 각 단계를 이해하고 싶은 분들을 위한 수동 설치 방법입니다.

### 3.1 `uv` 설치

  ```powershell
  irm https://astral.sh/uv/install.ps1 | iex
  ```

설치 후 새 셸을 열어 `uv --version`이 출력되는지 확인하세요.

### 3.2 저장소 클론

```bash
git clone https://github.com/sorryhyun/anima_lora.git
cd anima_lora
```

> 이 가이드는 모든 경로를 `anima_lora/` 디렉터리 기준으로 설명합니다. 모든 명령은 이 폴더 안에서 실행하세요.

### 3.3 의존성 설치

```bash
winget install ezwinports.make
uv sync
```

`uv sync`는 `pyproject.toml`/`uv.lock`을 보고 **`anima_lora/` 안에 `.venv/` 폴더(독립적인 파이썬 환경)를 만들어** 모든 의존성을 그 안에 설치합니다. 시스템 파이썬은 건드리지 않으므로 다른 프로젝트에 영향을 주지 않습니다 — 이것이 정상이며, 직접 `pip install` 을 할 필요는 없습니다.

> **초보자가 가장 많이 막히는 지점입니다.** `uv sync` 가 끝나면 패키지는 전역 파이썬이 아니라 `.venv/` 안에 들어 있습니다. 그냥 터미널을 열고 `python tasks.py ...` 나 `make lora` 를 실행하면, 그 셸은 여전히 *시스템* 파이썬을 쓰기 때문에 의존성이 없어 `ModuleNotFoundError` 가 나기 쉽습니다. 명령이 `.venv/` 의 파이썬을 가리키도록 해야 하며, 방법은 두 가지입니다.

**방법 A — `uv run` (활성화 불필요, 어디서나 동작).** 아무 명령 앞에 `uv run` 을 붙이면 자동으로 `.venv/` 를 씁니다:

```bash
uv run make lora              # 또는: uv run python tasks.py lora
uv run hf auth login
uv run make download-models
```

셸 상태에 의존하지 않으므로 환경을 "잊어버릴" 일이 없어 가장 확실한 방법입니다.

**방법 B — 터미널마다 한 번 venv 활성화.** 활성화하면 그 셸 세션 동안 `.venv/` 의 `python` 이 `PATH` 최우선이 되어, `uv run` 접두사 없이 `make ...` / `python tasks.py ...` 가 바로 동작합니다:

```powershell
.venv\Scripts\activate        # Windows (PowerShell / cmd)
```
```bash
source .venv/bin/activate     # Linux / macOS / WSL
```

프롬프트에 `(anima_lora)` (또는 `(.venv)`) 접두사가 보이면 성공입니다. **새 터미널 창마다 다시 활성화해야 합니다** — 활성화 상태는 유지되지 않습니다. 끝내려면 `deactivate`. VSCode 사용자는 `.venv` 인터프리터를 한 번 선택해두면(명령 팔레트 → *Python: Select Interpreter*) 통합 터미널이 자동으로 활성화됩니다.

> 이 가이드의 나머지 명령은 평범한 `make ...` / `python tasks.py ...` 형태로 적혀 있으며, venv 를 활성화했거나(방법 B) `uv run` 을 붙이는(방법 A) 것을 전제로 합니다. 명령이 `ModuleNotFoundError` 나 `command not found: make` 로 실패하면 거의 항상 이것이 원인입니다 — venv 를 활성화하거나 `uv run` 을 붙이세요.

## 5. 데이터셋 준비

Anima LoRA는 *이미지 + 같은 이름의 `.txt` 캡션 사이드카* 구조를 사용합니다. 아래는 `image_dataset/` 폴더의 예시입니다.

```
image_dataset/
├─ 00001.png
├─ 00001.txt
├─ 00002.jpg
├─ 00002.txt
├─ subfolder/
│  ├─ 00010.webp
│  └─ 00010.txt
└─ ...
```

### 5.1 캡션 작성 팁

- anima 공식 가이드라인을 따르면 태그 순서는 항상 [meta] [character] [series] [artist] [general] 입니다. 예를 들어

```
absurdres, safe, 1girl, chitanda eru, hyouka, @channel (caststation), full body, serafuku, She is saying hi.
```
와 같이 작성하시면 됩니다.

- 이 때 absurdres, highres, masterpiece 등의 퀄리티 태그는 개인적으로 실험한 결과 작성하지 않거나 최소한으로만 남기는 것이 제일 좋습니다. 혹은 추후 정식 배포될 mod guidance를 사용하면 아예 작성하지 않아도 됩니다.
- 원본 이미지는 `image_dataset/`에 둡니다 (이름은 자유, 이 위치를 그대로 사용하세요).

### 5.2 `num_repeats`는 무엇이고 언제 만지나요? (요약: **만지지 않습니다**)

`configs/base.toml`의 `[[datasets.subsets]]` 안에 `num_repeats = 1` 항목이 보일 겁니다. 이 값은 **한 에폭 동안 각 이미지를 몇 번 반복해서 사용할지**를 지정하는 kohya-ss 스타일 옵션으로, 다른 LoRA 트레이너 가이드를 따라하다 보면 자주 마주치는 키입니다.

- **이 가이드북의 표준 워크플로우에서는 그냥 `1`로 두세요.** 모든 이미지를 `image_dataset/` 한 폴더에 모아 학습하는 일반적인 사용 패턴에서는 `num_repeats`를 올려도 *한 에폭이 길어질 뿐* 효과는 `max_train_epochs`를 늘리는 것과 같습니다. 학습량은 항상 에폭 수로 조절하는 것이 직관적이며, 본 프로젝트의 모든 프리셋·메소드 설정은 `num_repeats = 1`을 전제로 튜닝되어 있습니다.
- **언제 올리는 게 의미가 있나요?** 한 학습에 *서로 다른 서브셋(폴더)이 여러 개* 있고 폴더별 이미지 수가 크게 차이날 때, 작은 폴더의 노출 빈도를 큰 폴더에 맞추기 위한 *밸런싱 도구*로만 의미가 있습니다 (예: 캐릭터 A 1000장 + 캐릭터 B 50장이라면 B 서브셋만 `num_repeats = 20`). 단일 폴더 학습에는 해당되지 않습니다.
- **어디서 바꾸나요?** `num_repeats`는 메소드 설정이 아니라 *데이터셋 설정*이라 `configs/methods/`·`configs/gui-methods/`·GUI 학습 탭에는 노출되지 않습니다. 정말 필요하다면 `configs/base.toml`의 `[[datasets.subsets]]`(또는 `--dataset_config <path>`로 따로 지정한 TOML)를 직접 편집하세요. *동일 이미지를 단순히 더 많이 학습시키고 싶다면* `num_repeats`가 아니라 `max_train_epochs`를 늘리는 것이 정답입니다.

## 6. 전처리: 리사이즈 · 잠재변수 · 텍스트 임베딩 캐싱

학습 속도와 VRAM을 최적화하기 위해 **리사이즈 → VAE 잠재 캐싱 → 텍스트 임베딩 캐싱** 세 단계를 사전 수행합니다.

```bash
make preprocess              # 세 단계 모두 실행 (LoRA / 일반 학습용)
# 또는 단계별로
make preprocess-resize       # 1) image_dataset/ → post_image_dataset/resized/
make preprocess-vae          # 2) VAE 잠재변수 캐싱 → post_image_dataset/lora/
make preprocess-te           # 3) 텍스트 인코더 출력 캐싱 → post_image_dataset/lora/
make preprocess-pe           # (선택) PE-Core 비전 인코더 피처 캐싱 — IP-Adapter 전용
```

> **⚠️ 캐시는 재사용됩니다 — 절대 자동 삭제되지 않습니다.**
> `make preprocess`(그리고 GUI의 *Preprocess* 버튼)는 **이미 만들어둔 캐시를 그대로 재사용**합니다. `post_image_dataset/lora/` 안의 `.npz` / `_te.safetensors` / `_pe.safetensors` 파일은 *덮어쓰지도, 지우지도 않으며*, 누락된 항목만 새로 처리합니다. 그래서 재실행이 매우 빠르고, 도중에 끊겨도 안전합니다.
>
> 즉, 이미 캐시가 있는 상태에서 `make preprocess`를 다시 돌려도 기존 데이터가 사라지지 않으니 안심하세요. 반대로 **캡션 / 토크나이저 / 리사이즈 옵션을 바꿔서 *처음부터* 다시 만들고 싶다면**, 캐시 폴더(`post_image_dataset/lora/` 또는 `post_image_dataset/easycontrol/`)를 직접 지우고 다시 실행해야 합니다.

### 6.1 리사이즈가 하는 일

- VAE가 요구하는 픽셀 정렬에 맞춰 이미지를 리사이즈
- *(H/16) × (W/16) = 4032 또는 4200 패치*인 두 가지 토큰 수 계열의 *고정 토큰 버킷*에 자동 분류 (각 버킷은 해당 토큰 수를 정확히 채움)
- 너무 작은 이미지(기본 0.5MP 미만)는 자동 제외하고 보고
- 결과는 `post_image_dataset/resized/`에 PNG로 저장

### 6.2 잠재변수 캐싱

- 모든 리사이즈 이미지에 대해 VAE를 한 번만 돌려 결과를 디스크에 저장
- 학습 중에는 VAE를 GPU에 올리지 않으므로 VRAM이 크게 절약됨
- 캐시 위치: `post_image_dataset/lora/{stem}_{WxH}_anima.npz`
- 스크립트: `scripts/preprocess/cache_latents.py`

### 6.3 텍스트 임베딩 캐싱

- Qwen3 0.6B + LLM 어댑터 출력을 미리 계산
- `use_shuffled_caption_variants = true`이면 캡션 콤마 셔플 변형까지 같이 캐싱 (학습 시 무작위 선택)
- 캐시 위치: `post_image_dataset/lora/{stem}_anima_te.safetensors`
- 캡션은 항상 `image_dataset/`의 원본 `.txt`를 읽습니다 (리사이즈 폴더에 복사하지 않음)
- 스크립트: `scripts/preprocess/cache_text_embeddings.py`

### 6.4 PE 비전 피처 캐싱 (선택)

- IP-Adapter 학습(및 DCW v4 fusion head)에만 필요
- PE-Core-L14-336 비전 인코더 출력을 미리 계산해 학습 중 비전 인코더를 올리지 않도록 함
- 캐시 위치: `post_image_dataset/lora/{stem}_anima_pe.safetensors`

> **캐시는 언제 다시 만들어야 하나요?**
> - 이미지가 추가되었을 때 → 그냥 `make preprocess`를 다시 실행하면 됩니다 (기존 캐시는 유지되고 새 항목만 추가됩니다).
> - **캡션을 수정**했거나 **토크나이저/패딩 관련 옵션을 바꿨을 때** → 위 ⚠️ 안내대로 캐시 폴더(`post_image_dataset/lora/`)를 직접 지운 뒤 다시 실행하세요. 단순 재실행은 *기존 캐시를 그대로 재사용하므로* 변경 사항이 반영되지 않습니다.

## 7. GUI 사용법

PySide6 기반 GUI를 사용하면 설정 편집, 데이터셋 탐색, 전처리, 학습 실행/모니터링, LoRA 병합을 한 화면에서 할 수 있습니다.

```bash
make gui
make gui-shortcut   # (선택) 윈도우 바탕화면에 콘솔 창 없는 .lnk 바로가기 생성
```

GUI 주요 탭:

- **학습 설정 (Training Config)**: LoRA 계열 변형(권장: `tlora` — Ortho + T-LoRA / 그 외 `lora`, `tlora-8gb`, `tlora_ortho_reft`, `hydralora`, `reft` 등)을 드롭다운으로 선택하고, `presets.toml` 프리셋(default / low_vram 등)과 모든 학습 키를 직접 수정해 그대로 학습 시작
- **전처리 (Preprocess)**: 리사이즈 + VAE + 텍스트 임베딩 캐싱을 한 번에
- **데이터셋 (Dataset)**: 이미지/캡션을 미리보고 캡션을 직접 수정
- **병합 (Merge)**: 학습된 LoRA를 베이스 DiT에 베이크해 ComfyUI 단독 체크포인트로 저장 (기본 LoRA / OrthoLoRA / T-LoRA만 지원)

GUI 학습도 결국 내부적으로 `train.py`를 호출하므로, 같은 파라미터를 CLI에서도 그대로 재현할 수 있습니다. GUI는 `configs/gui-methods/<variant>.toml`(토글 블록 없는 변형별 단일 파일)을 읽으므로, GUI에서 노출되는 변형 목록과 CLI의 `make lora-gui GUI_PRESETS=<variant>`가 같은 결과를 냅니다. 변형 목록의 현재 상태는 `ls configs/gui-methods/` 로 확인하세요.

### 7.1 폼 편집과 저장 동작

학습/전처리 서브프로세스는 디스크의 variant TOML 파일을 다시 읽어들이기 때문에, 폼만 수정하고 저장하지 않으면 그 변경 사항은 학습에 반영되지 않습니다. GUI는 이를 두 가지로 처리합니다.

- **변경 감지**: 폼의 어떤 필드라도 (또는 `+ Extra args` 텍스트박스를) 편집하면 `Save` 버튼이 주황색으로 바뀌고 `Save *` 표시가 붙습니다. 즉 *디스크의 variant 파일과 화면이 다르다*는 신호입니다. `Save`를 누르거나 variant를 다시 선택해서 디스크에서 다시 불러오면 표시가 사라집니다.
- **자동 저장**: 그래도 저장을 깜빡하고 `Train` / `Preprocess`를 누르면, 해당 시점의 폼 값이 자동으로 variant 파일에 먼저 기록된 뒤 서브프로세스가 실행됩니다. 즉 화면에 보이는 값이 곧 학습에 쓰이는 값입니다. (`Test`는 마지막 학습 결과 체크포인트로 추론하므로 자동 저장 대상이 아닙니다.)

> 변경 사항을 *시험만* 해보고 싶다면, 폼을 편집한 뒤 `Train`을 누르지 말고 variant를 한 번 다른 것으로 바꿨다 돌아오세요 — 디스크에서 다시 로드되어 편집이 폐기됩니다.

### 7.2 자동 재개 (checkpointing_epochs)

학습이 도중에 끊겨도 다시 `Train`을 누르면 **자동으로 마지막 저장 지점부터 이어 학습**합니다. 정전·OOM·실수로 창 닫기 등에 대비할 수 있는 가장 유용한 기능 중 하나이므로 기본값으로 켜져 있습니다.

GUI에서 다루는 방법:

- 학습 설정 탭의 **Training 그룹**에 `checkpointing_epochs` 필드가 있습니다 (gui-methods 변형 기본값 `2`, `methods/lora.toml` 기본값 `4`). N 에폭마다 재개용 상태를 저장하며, 같은 파일을 덮어쓰므로 디스크가 부풀지 않습니다.
- 학습이 끊긴 뒤 같은 variant로 다시 `Train`을 누르면, 로그 창에 `auto-resuming from checkpoint at step N`이 뜨고 그 지점부터 이어 학습합니다. 별도 플래그를 손댈 필요 없습니다.
- 학습이 정상적으로 끝나면 재개용 임시 파일은 자동 삭제되고, 최종 결과는 `output/ckpt/<output_name>.safetensors`로 남습니다.
- **데이터셋이나 핵심 설정(랭크·LR·에폭 수 등)을 바꾼 뒤** 새로 학습하려면 `output/ckpt/<output_name>-checkpoint-state/` 폴더를 수동으로 지운 다음 `Train`을 누르세요. 그렇지 않으면 옛 상태 위에 학습이 이어집니다.

상세 동작은 [§8.6 자동 재개](#86-자동-재개-checkpointing_epochs)에서 다룹니다 — `save_every_n_epochs`와의 차이점도 거기서 정리.

### 7.3 학습 중단과 GUI 닫기

학습은 GUI 창 *안에서* 도는 것이 아닙니다 — `Train`을 누르면 작업은 `train.py`를 분리된(detached) 프로세스로 실행하는 작은 백그라운드 **학습 데몬**에 넘겨집니다. 여기서 실용적인 결과가 두 가지 나옵니다:

- **`Stop` 버튼은 현재 학습 작업을 중단합니다.** 데몬은 계속 떠 있으면서 (큐에 있다면) 다음 작업으로 넘어가므로, 한 작업을 멈춰도 큐 전체가 무너지지 않습니다. CLI의 `make daemon-kill`과 같습니다. (`Stop`은 진행 중인 `Test`나 `Preprocess`도 취소하는데, 이 둘은 GUI *안에서* 돕니다.)
- **GUI를 닫아도 학습은 멈추지 않습니다.** 작업이 분리된 데몬에서 돌기 때문에 창을 닫아도 학습은 계속됩니다 — 밤새 돌려두고 싶은 긴 학습에 유용합니다. GUI를 다시 열면(`make gui`) 아직 도는 작업에 자동으로 다시 연결되며, 로그에 `Re-attached to running job …`이 보이고 진행 바와 출력이 실시간으로 이어집니다. (CLI나 ComfyUI 트레이너 노드에서 시작한 작업도 이렇게 드러납니다.)

> 학습을 완전히 끄려면 — 진행 중인 작업을 죽이고 *동시에* 데몬도 멈춰 GPU를 풀려면 — CLI에서 `make daemon-terminate`를 쓰세요. `Stop`만으로는 데몬이 계속 떠 있습니다.
>
> `Test`와 `Preprocess`는 예외입니다: 창 안의 서브프로세스로 돌기 때문에 GUI를 닫으면 취소됩니다.

## 8. 학습 실행

모든 학습은 TOML 설정 파일과 HuggingFace Accelerate를 통해 실행됩니다. 설정 병합 순서는 `configs/base.toml → configs/presets.toml[<preset>] → configs/methods/<method>.toml → CLI args`이며, 메소드 설정이 프리셋을 덮어씁니다.

### 8.1 빠른 시작

**가장 추천하는 출발점은 OrthoLoRA + T-LoRA (=`tlora` 변형)** 입니다. 안정성·디테일·스타일 보존이 가장 균형 잡힌 조합으로, 보통의 캐릭터/스타일 LoRA 학습에 그대로 쓸 수 있습니다.

```bash
# 권장: Ortho + T-LoRA (gui-methods/tlora.toml)
make lora-gui GUI_PRESETS=tlora                  # 일반 환경
PRESET=low_vram make lora-gui GUI_PRESETS=tlora-8gb   # VRAM 8~12GB

# 기타 변형 (configs/gui-methods/<variant>.toml — 토글 블록 없는 단일 파일)
make lora-gui GUI_PRESETS=lora                   # 평범한 LoRA만
make lora-gui GUI_PRESETS=tlora_ortho_reft       # Ortho + T-LoRA + ReFT 스택
make lora-gui GUI_PRESETS=hydralora              # MoE 다중 헤드
make lora-gui GUI_PRESETS=reft                   # ReFT 단독

# 토글 블록 방식 (configs/methods/lora.toml 안에서 직접 변형 선택)
make lora                          # presets.toml[default]
PRESET=low_vram make lora          # presets.toml[low_vram] — VRAM 8~12GB
PRESET=half make lora              # 데이터셋의 절반만 사용해 빠른 실험
```

> **CLI에서 직접 키 오버라이드**: `make lora -- --network_dim 32 --max_train_epochs 24` 처럼 추가 인자를 그대로 전달할 수 있습니다 (`tasks.py`도 동일).

### 8.2 마스크드 로스 (텍스트 버블 제외)

만화/만화풍 데이터에서 *말풍선이나 텍스트 영역*을 학습 손실에서 제외하면 결과가 훨씬 깨끗해집니다.

```bash
make mask          # SAM3 + MIT (임시 디렉토리에서 실행) → post_image_dataset/masks/
make mask-clean    # post_image_dataset/masks/ 삭제
```

결과 PNG는 흑백입니다: **흰색(255) = 학습 대상**, **검정(0) = 제외**. 데이터셋 서브셋은 `post_image_dataset/masks/`가 있으면 그것을 자동으로 사용하며, 없으면 레거시 `masks/{merged,sam,mit}/` 를 순서대로 fallback합니다 (이전 레이아웃을 그대로 쓰는 사용자도 작동). 마스크 파일이 없으면 그냥 무시되므로 안 만들어도 됩니다.

### 8.3 자주 만지는 설정값 (LoRA 기준 기본값)

| 파라미터 | 기본값 | 설명 |
|---|---|---|
| `network_dim` | `32` | LoRA 랭크. 클수록 표현력↑, 파라미터↑ |
| `network_alpha` | `32` | LoRA 스케일 (보통 `network_dim`과 같은 값) |
| `learning_rate` | `2e-5` | 학습률. Hydra는 더 낮춰도 됨 |
| `max_train_epochs` | `4` | 데이터셋이 작을수록 에폭↑ |
| `save_every_n_epochs` | `2` (gui-methods) / `4` (methods) | 어댑터 가중치 누적 저장 주기 |
| `checkpointing_epochs` | `2` (gui-methods) / `4` (methods) | 재개용 상태 저장 주기 (단일 파일 덮어쓰기) |
| `caption_dropout_rate` | `0.1` | 캡션 일부를 빈 문자열로 대체 (CFG에 도움) |
| `use_shuffled_caption_variants` | `true` | 캡션 콤마 셔플 변형 사용 |

각 변형 토글(`use_ortho`, `use_timestep_mask`, `add_reft`, `use_moe_style`, `router_source` 등)은 `configs/methods/lora.toml`의 주석 블록을 해제해 활성화하거나, `configs/gui-methods/<variant>.toml`에서 변형별 단일 파일을 그대로 사용하세요. **추천 출발점인 `tlora` 변형은 `use_ortho = true` + `use_timestep_mask = true` 가 미리 켜져 있는 OrthoLoRA + T-LoRA 조합입니다.**

### 8.4 학습 진행 중 일어나는 일

1. 텍스트 인코더 로드 → 캐시 생성/검증 → 언로드
2. VAE 로드 → 캐시 생성/검증 → 언로드
3. DiT를 *지연 로딩*하여 캐싱 단계의 VRAM 충돌 방지
4. 어댑터 네트워크를 DiT의 attention / FFN 모듈에 패치 (변형마다 대상이 다름)
5. 노이즈 샘플링 → DiT forward → flow-matching 손실 → backward → optimizer step
6. (선택) `validation_split`로 검증 손실 측정 및 샘플 이미지 생성

### 8.5 출력물

- 학습된 가중치: `output/ckpt/<output_name>.safetensors` (변형마다 `anima`, `anima_tlora_ortho`, `anima_tlora_reft`, `anima_hydra`, `anima_postfix` 등으로 자동 분기)
- 체크포인트: `save_every_n_epochs` 주기마다 `output/ckpt/`에 저장 (`.snapshot.toml` 사이드카 + Hydra의 경우 `_moe` 동반 파일)
- 검증 샘플: `output/ckpt/sample/`
- 추론 결과 이미지: `output/tests/`

### 8.6 자동 재개 (checkpointing_epochs)

학습이 도중에 끊기더라도 **마지막으로 저장된 지점부터 자동으로 이어서** 재개할 수 있는 기능입니다. 정전, OOM, 실수로 창 닫기, 잠깐 PC 끄고 자고 싶을 때 등에 매우 유용합니다. 기본 메소드 파일에 이미 켜져 있으므로 따로 설정할 필요는 없습니다.

```toml
checkpointing_epochs = 2     # 2 에폭마다 재개용 상태 저장 (덮어쓰기)
```

`save_every_n_epochs`와는 역할이 다릅니다.

| 키 | 저장 대상 | 누적? | 용도 |
|---|---|---|---|
| `save_every_n_epochs` | 어댑터 가중치 (`anima_lora-000004.safetensors` 등) | **누적** (또는 `save_last_n_epochs`로 제한) | 중간 결과로 추론을 돌려 보거나 과적합 시점을 비교 |
| `checkpointing_epochs` | 학습 재개용 풀 상태 (옵티마이저 / 스케줄러 / RNG / 어댑터 가중치) | **단일 파일을 덮어씀** | 학습 중단 시 자동 이어 학습 |

동작 방식:

- **자동 저장**: `checkpointing_epochs` 주기로 `output/ckpt/<output_name>-checkpoint-state/`(상태 디렉터리) + `<output_name>-checkpoint.safetensors`(가중치)가 함께 저장되며, 이전 것을 덮어씁니다. 디스크가 부풀지 않습니다.
- **자동 재개**: 같은 명령(`make lora` 등)을 다시 실행하면, 저장된 체크포인트가 있고 아직 `max_train_steps`에 도달하지 못했다면 **알아서 이어 학습합니다**. `--resume` 같은 플래그를 직접 줄 필요 없음. 로그에 `auto-resuming from checkpoint at step N`이 보이면 정상 재개된 것.
- **자동 정리**: 학습이 정상적으로 끝나면 위 두 파일이 자동 삭제됩니다 — 재개용 상태는 임시 파일이며, 최종 결과물은 `output/ckpt/<output_name>.safetensors`(8.5 참고)입니다.
- **수동 재개**: 다른 시점의 상태로 돌아가고 싶다면 `--resume <state_dir>`를 직접 지정해도 됩니다.

> **언제 비활성화?** 짧은 실험 학습이나 디스크가 부족한 경우 `checkpointing_epochs`를 주석 처리하면 됩니다. 데이터셋이 큰 본 학습에서는 거의 항상 켜두는 것을 권장합니다.
>
> **주의**: 데이터셋, 캡션, 학습 설정(랭크·LR·에폭 수 등)을 바꿨다면 이전 체크포인트로 재개하는 것이 의미가 없거나 위험합니다. 이 경우 `output/ckpt/<output_name>-checkpoint-state/` 폴더를 수동으로 지우고 새로 시작하세요.

## 9. LoRA / 어댑터 변형 선택 가이드

> **🌟 추천**: 처음이거나 일반적인 캐릭터·스타일 LoRA를 만들 거라면 **`tlora` (OrthoLoRA + T-LoRA)** 로 시작하세요. 디테일/스타일 균형과 학습 안정성이 가장 좋습니다.

| 변형 | 실행 방법 | 언제 쓰나요? |
|---|---|---|
| **OrthoLoRA + T-LoRA** ⭐ | `make lora-gui GUI_PRESETS=tlora` | **추천**. SVD 기반 직교 회전(OrthoLoRA) + 타임스텝별 랭크 마스킹(T-LoRA) 스택. `anima_tlora_ortho.safetensors` 산출 |
| **OrthoLoRA + T-LoRA (8GB)** | `make lora-gui GUI_PRESETS=tlora-8gb` 또는 `PRESET=low_vram make lora-gui GUI_PRESETS=tlora` | VRAM 8~12GB 환경에서 위 추천 조합을 그대로 사용 |
| **평범한 LoRA** | `make lora-gui GUI_PRESETS=lora` 또는 `make lora` | 가장 단순한 베이스라인, 비교 실험용 |
| **평범한 LoRA (8GB)** | `make lora-gui GUI_PRESETS=lora-8gb` 또는 `PRESET=low_vram make lora` | VRAM 8~12GB |
| **T-LoRA + Ortho + ReFT** | `make lora-gui GUI_PRESETS=tlora_ortho_reft` | 위 추천 조합에 표현 편집(ReFT)을 더해 더 적은 추가 파라미터로 미세 조정 |
| **HydraLoRA** | `make lora-gui GUI_PRESETS=hydralora` (8GB는 `hydralora-8gb`) | MoE 다중 헤드 라우팅, 다양한 콘셉트를 한 어댑터에 |
| **ReFT 단독** | `make lora-gui GUI_PRESETS=reft` 또는 `methods/lora.toml`에서 `add_reft = true` | 표현 편집(Representation Fine-Tuning), 매우 적은 파라미터 |
| **Postfix Tuning** *(실험적)* | `make exp-postfix` 또는 `make lora-gui GUI_PRESETS=postfix_ortho_cond` | cross-attention 끝에 학습 가능한 N개 벡터 (캡션 조건 + 직교 변형) |
| **ChimeraHydra** *(실험적)* | `make exp-chimera` 또는 `make lora-gui GUI_PRESETS=chimera_hydra` | 콘텐츠/주파수 듀얼 풀 MoE — 연구용 |

각 변형의 자세한 옵션은 [`docs/guidelines/training.md`](training.md) 및 `docs/methods/`의 개별 문서를 참고하세요.

> **호환성 메모**
> - HydraLoRA / ReFT 등 어댑터 변형은 `cache_llm_adapter_outputs = true`가 기본으로 켜져 있어야 정상 동작합니다.
> - `tlora` 와 `tlora_ortho_reft` 의 OrthoLoRA + T-LoRA 부분은 `make merge` 로 베이스 DiT에 베이크해 단독 ComfyUI 체크포인트로 만들 수 있습니다 (ReFT 부분은 베이크 불가 — `--allow-partial` 필요).

## 10. 추론

### 10.1 가장 빠른 테스트

방금 학습한 어댑터로 즉시 샘플을 만들고 싶다면, 변형에 맞는 `make test-*`를 쓰세요. 모두 `output/ckpt/`에서 가장 최근에 저장된 해당 어댑터를 자동으로 골라 추론합니다.

```bash
make test                        # 일반 LoRA / OrthoLoRA / T-LoRA / ReFT
make test SPECTRUM=1             # Spectrum 가속 추론
make test MOD=1                  # Modulation guidance (pooled_text_proj) — SPECTRUM=1과 조합 가능
make test NOLORA=1               # 베이스 DiT 단독 추론 (--lora_weight 생략); MOD=1과 합치면 mod-only 경로
make test-hydra                  # HydraLoRA (router-live, anima_hydra*_moe.safetensors)
make test-merge                  # 베이크된 단독 DiT(`*_merged.safetensors`)로 추론
make test-dcw                    # LoRA + DCW 스칼라 보정 (샘플러 단계 SNR-t 보정)
make test-dcw-v4                 # LoRA + DCW v4 학습형 보정기
# 실험적 추론
make exp-test-postfix            # Postfix tuning (vanilla)
make exp-test-postfix-exp        # postfix_exp 변형
make exp-test-postfix-func       # postfix_func 변형
```

### 10.2 일반 추론 (수동)

```bash
python inference.py \
    --dit models/diffusion_models/anima-base-v1.0.safetensors \
    --text_encoder models/text_encoders/qwen_3_06b_base.safetensors \
    --vae models/vae/qwen_image_vae.safetensors \
    --lora_weight output/ckpt/anima_lora.safetensors \
    --lora_multiplier 1.0 \
    --prompt "masterpiece, best quality, an anime girl in a sunlit forest" \
    --negative_prompt "worst quality, low quality, blurry" \
    --image_size 1024 1024 \
    --infer_steps 30 \
    --guidance_scale 4.0 \
    --sampler er_sde \
    --flow_shift 1.0 \
    --seed 42 \
    --save_path output/tests
```

자주 쓰는 플래그:

| 플래그 | 설명 |
|---|---|
| `--lora_weight` | 학습된 어댑터 경로. 여러 개 동시에 줄 수도 있음 |
| `--lora_multiplier` | 어댑터 강도 (0.0~1.5) |
| `--image_size H W` | 출력 해상도 (예: `1024 1024`, `1024 1536`) |
| `--infer_steps` | 디노이징 스텝 수 (보통 20~50) |
| `--guidance_scale` | CFG 강도 (3.0~5.0 권장) |
| `--sampler` | `er_sde`, `euler`, `dpm++` 등 |
| `--seed` | 재현용 시드 |
| `--spectrum` | Spectrum 가속 활성화 |
| `--pgraft` | P-GRAFT (디노이즈 후반 LoRA 컷오프)로 베이스 모델이 후반 디테일 담당 |

추론 옵션의 전체 목록과 P-GRAFT 추론 방법은 [`docs/guidelines/inference.md`](inference.md)에 있습니다.

## 11. ComfyUI로 배포

ComfyUI 코어는 Anima 베이스 DiT를 기본 지원합니다 (`UNETLoader` / `CLIPLoader`로 그대로 로드). 어댑터 종류에 따라 배포 방식이 다릅니다.

### 11.1 클래식 LoRA / OrthoLoRA / T-LoRA

`output/ckpt/`에서 나온 `.safetensors`를 그대로 `ComfyUI/models/loras/`에 복사하면, ComfyUI 기본 LoraLoader 노드로 바로 사용할 수 있습니다. 더 깔끔하게 단독 체크포인트로 만들고 싶다면:

```bash
make merge ADAPTER_DIR=output/ckpt                 # 가장 최근 가중치를 베이스 DiT에 베이크
make merge ADAPTER_DIR=output/ckpt MULTIPLIER=0.8  # 강도 조절
```

베이크된 `*_merged.safetensors`는 그대로 ComfyUI의 `UNETLoader`에서 단독 모델로 로드 가능합니다.

### 11.2 HydraLoRA / ReFT / Postfix

이 변형들은 ComfyUI 기본 LoraLoader로 로드할 수 없고 (단순 weight delta가 아니라 라우팅·토큰 삽입이 들어감), 전용 노드가 필요합니다:

- **Anima Adapter Loader** (`custom_nodes/comfyui-hydralora/`) — LoRA / Hydra / ReFT / postfix를 통합 처리. 자세한 사용법은 해당 폴더의 `README.md` 참고.
- **Spectrum KSampler / Mod Guidance / DCW 노드** — 별도 저장소 <https://github.com/sorryhyun/ComfyUI-Spectrum-KSampler>
