# `per_channel_scaling` calibration is content-agnostic on Anima

This records an A/B result showing that the per-input-channel activation
profile `channel_stats.safetensors` captures — the thing `per_channel_scaling`
rescales LoRA inputs against — is **driven by the model's weights/architecture,
not by image content**. Real photo latents and teacher-clean synthetic latents
produce essentially the same per-channel dominance structure. The practical
consequences: the shipped calibration generalizes across datasets (no
per-dataset recompute needed), and the `distill_mod_synth` teacher pool is
**redundant for calibration** (a dead-end reuse direction).

Method/mechanism reference: `bench/channel_stats/` and the audit memory
`project_per_channel_scaling_audit`. Calibration consumer:
`networks/lora_modules/base.py::_absorb_channel_scale` (gated by
`channel_scaling_alpha > 0`).

## What was tested

The calibration metric is per-input-channel **dominance** =
`mean|x|.max / median` across the channels feeding each Linear, accumulated over
forward passes at several flow-matching sigmas. High dominance = a few channels
carry outsized magnitude (DC-bias-like), which is what `per_channel_scaling`
sqrt/flattens via `s_norm`.

The A/B held everything constant except the **latent content**:

- **Real side** — each stem's real VAE latent (`post_image_dataset/lora/`).
- **Synth side** — the same stem's *teacher-clean* latent from the
  `distill_mod_synth` pool (frozen base Anima, CFG=2.5, 24-step output;
  generated by `make distill-prep`).

Both runs used the **same 1823 stems, same TE sidecars, same sigmas**
(`0.1,0.3,0.5,0.7,0.9`), `--per_artist` (71 samples, one per artist) × 5σ =
355 forwards each, on `anima-base-v1.0.safetensors`, eager. The two temp dirs
symlinked the same TE sidecar next to either the real or the synth `npz` so the
analyzer's stem→latent+TE pairing resolved identically on both sides.

Run: `bench/channel_stats/analyze_lora_input_channels.py --dataset_dir <dir>
--per_artist --out_json <out>`, then diffed `overall` / `groups` / `per_module`.

## Result — the profiles are the same

| Check | Result |
|---|---|
| Same dominant channel per module (argmax) | **309 / 314 (98%)** |
| Per-module `dominance_ratio` drift (synth vs real) | median **0.2%**, p90 **3.3%**, max 11.7% |
| Top-channels set overlap (Jaccard) | median **1.00** |
| Overall dominance (mean / median / p90) | identical to ~1% |
| Overall p99 / max dominance | **bit-identical** (117.31 / 648.68) |

By module group (mean dominance, synth/real ratio):

| Group | real | synth | s/r |
|---|---|---|---|
| `mlp.layer1_in` | 37.61 | 38.04 | 1.01 |
| `cross_attn.q_in` | 30.07 | 30.59 | 1.02 |
| `self_attn.qkv_in` | 23.36 | 23.47 | 1.00 |
| `other` | 17.31 | 17.31 | 1.00 |
| `cross_attn.out_in` | 11.52 | 11.34 | 0.98 |
| `mlp.layer2_in` | 10.12 | 10.02 | 0.99 |
| `self_attn.out_in` | 8.60 | 8.67 | 1.01 |
| `cross_attn.kv_in` | 1.83 | 1.83 | 1.00 |

The few-percent residual drift is concentrated in MLP-layer2 and cross-attn-Q
(worst single module 12%), and **never flips the dominant channel**.
`cross_attn.kv_in` is flat at 1.83 on both sides — expected, since that input is
fed by the text embedding, which was identical across the two runs.

## Why this is the expected outcome in hindsight

The dominant channels are **DC-bias outliers** (peak-to-mean on the top channel
is ~1.0–1.5 for most modules: a near-constant offset across all tokens), not
content-modulated spikes. A near-constant per-channel offset is a property of
the model's learned weights/biases, so it shows up whether the latent is a real
photo or a denoised teacher sample. This matches the prior audit note that
"sigma coverage matters more than sample count because the dominant channels are
DC-bias (~constant across tokens)" — content is simply not a variable the
metric is sensitive to.

## Implications

1. **The shipped `networks/calibration/channel_stats.safetensors` generalizes.**
   It is not overfit to a particular artist set or to the real-photo
   distribution; recomputing it per-dataset is unnecessary. Re-run it only after
   changes that alter the *activations themselves* — model weights, latent
   caching/bucketing — not when the training images change.

2. **The teacher synth pool is a dead end for calibration.** Clean teacher
   latents are redundant with real latents here. The pool's real ROI is
   prior-preservation regularization for identity/style LoRA — see
   `docs/proposal/prior_preservation_from_synth_pool.md`.

## Reproduce

Build matched real/synth dirs (symlink each synth `npz` + the stem's TE sidecar
into one tree, the real `npz` + same TE into another), then:

```bash
uv run python bench/channel_stats/analyze_lora_input_channels.py \
  --dataset_dir <real_dir>  --per_artist --out_json /tmp/cs_real.json
uv run python bench/channel_stats/analyze_lora_input_channels.py \
  --dataset_dir <synth_dir> --per_artist --out_json /tmp/cs_synth.json
# diff overall/groups/per_module: dominant-channel argmax agreement +
# per-module dominance_ratio relative drift.
```
