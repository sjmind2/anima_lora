FeRA

著者忠実な FeRA (Yin et al., arXiv:2511.17979)。Independent-A スタックドエキスパート — 各エキスパートが独自の lora_down/lora_up を持つ — HydraLoRA が使用する per-layer アクティベーションルーターではなく、周波数エネルギーインデックス FEI(z_t) によってフィードされる単一グローバルルーターでゲーティングされます。三軸設定 use_moe_style="independent_A" + route_per_layer=false + router_source="fei" を通じて LoRA ファミリーネットワークモジュールに組み込まれています。

各エキスパートは OrthoLoRA パラメタライズ (use_ortho=true) されており、per-expert (down, up) ファクターが per-expert Cayley 回転 + 対角 λ を持つ PSOFT スタイルの SVD 基底を共有します。FECL (周波数エネルギー一貫性ロス) は接続されていますがデフォルト無効 — 有効化すると 2 回目のノーグラード DiT フォワードによりステップごとのコストが 2 倍になります。

ベンチノート。 論文の 3 バンド分割は Anima flow-matching 潜在変数では約 2 バンドに崩壊します (中間バンドはエネルギー <8%)。デフォルトは著者忠実性のために fera_num_bands=3 を保持します; Anima で検証されたバンド数では fera_fecl_weight=0 で 2 に切り替えられます。σ_low = min(H_lat, W_lat) / fei_sigma_low_div ルール (デフォルト div=4) は Anima 向けにベンチ検証済みです — 論文のピクセルドメイン min(H, W)/128 は SD2-512² 固有です。

学習すると anima_fera*.safetensors + グローバルルーターを保持する *_moe.safetensors サイドカーが生成されます。ルーターライブ推論は make test-hydra / ComfyUI の Anima Adapter Loader を使用します (HydraLoRA と同じスニッフパス)。cache_llm_adapter_outputs=true が必要です。
