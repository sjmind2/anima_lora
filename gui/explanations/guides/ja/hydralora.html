HydraLoRA

LoRA の上に MoE スタイルのルーティング: 共有 lora_down + num_experts 個の per-expert lora_up ヘッド。適応された Linear の入力からレイヤーローカルでルーティングされます。負荷分散ロス (balance_loss_weight) でルーターが単一エキスパートに崩壊するのを防ぎます。

正弦波(σ)→E のルーターバイアスと σ バンドへの強制特化を追加します: specialize_experts_by_sigma_buckets = true でエキスパートプールをタイムステップバケットで分割し、各 σ バンドが割り当てられたエキスパートにのみルーティングします (例: num_experts = 6 と 3 バケットで、バンドごとに 2 エキスパート)。sigma_bucket_boundaries = [0.0, 0.5, 0.8, 1.0] で低ノイズの後期精製レジームに多く割り当てます。

バリアント
• hydralora — デフォルト 16GB 以上のプロファイル。
• hydralora-8gb — 同じアーキテクチャの低 VRAM プロファイル (勾配チェックポインティング + オフロード); デフォルトで OOM になる場合はこちらを選択してください。

学習するとアダプターの隣に *_moe.safetensors サイドカーが生成されます。ルーターライブ推論には両方のファイルが必要です (make test-hydra / ComfyUI の Anima Adapter Loader)。cache_llm_adapter_outputs = true が必要です。
