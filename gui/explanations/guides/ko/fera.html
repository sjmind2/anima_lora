FeRA

저자 충실 FeRA (Yin et al., arXiv:2511.17979). Independent-A 스택드 전문가 — 각 전문가가 자체 lora_down/lora_up을 가지며 — 단일 글로벌 라우터가 주파수-에너지 지수 FEI(z_t)를 입력으로 받아 게이팅합니다 (HydraLoRA의 레이어별 활성화 라우터와 대조). LoRA 계열 네트워크 모듈에서 3축 설정 use_moe_style="independent_A" + route_per_layer=false + router_source="fei"로 선택됩니다.

각 전문가는 OrthoLoRA 파라미터화 (use_ortho=true) — 전문가별 (down, up)이 PSOFT 스타일 SVD 베이시스를 공유하며 전문가별 Cayley 회전 + 대각 λ를 가집니다. FECL (주파수-에너지 일관성 손실)은 배선되어 있지만 기본 비활성 — 활성화하면 두 번째 no-grad DiT 포워드 때문에 스텝당 비용이 ~2배가 됩니다.

벤치 노트. 논문의 3밴드 분할은 Anima flow-matching 잠재공간에서 ~2밴드로 수렴합니다 (중간 밴드 <8% 에너지). 기본값은 저자 충실성을 위해 fera_num_bands=3; Anima-검증 분할은 fera_fecl_weight=0과 함께 2로 설정. σ_low = min(H_lat, W_lat) / fei_sigma_low_div 규칙 (기본 div=4)은 Anima에서 벤치-검증; 논문의 픽셀 도메인 min(H, W)/128은 SD2-512² 전용입니다.

학습 시 anima_fera*.safetensors + 글로벌 라우터를 담은 *_moe.safetensors 동반 파일이 생성됩니다. 라우터-라이브 추론은 make test-hydra / ComfyUI Anima Adapter Loader 사용 (HydraLoRA와 동일 sniff 경로). cache_llm_adapter_outputs=true 필요.
