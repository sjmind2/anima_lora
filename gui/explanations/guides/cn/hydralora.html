HydraLoRA

在 LoRA 上叠加 MoE 风格路由：共享 lora_down + num_experts 个专家各自的 lora_up 头，由适配 Linear 的输入在层本地进行路由。负载均衡损失（balance_loss_weight）防止路由器坍缩到单一专家。

增加 sinusoidal(σ)→E 路由器偏置与硬 σ 频带专业化：specialize_experts_by_sigma_buckets = true 将专家池按时间步桶分区，使每个 σ 频带只路由到其分配的专家（例如 num_experts = 6、3 个桶时每频带 2 个专家）。sigma_bucket_boundaries = [0.0, 0.5, 0.8, 1.0] 不均匀分布分割点 — 为后期低噪声细化阶段分配更多容量。

变体
• hydralora — 默认 16GB+ 配置。
• hydralora-8gb — 相同架构，低显存配置（梯度检查点 + 卸载）；如果默认配置 OOM，请选此变体。

训练会在适配器旁生成 *_moe.safetensors 附属文件；路由器实时推理需要两个文件（make test-hydra / ComfyUI Anima Adapter Loader）。需要 cache_llm_adapter_outputs = true。
