HydraLoRA

MoE-style routing on top of LoRA: shared lora_down + num_experts per-expert lora_up heads, routed layer-locally from the adapted Linear's input. A load-balance loss (balance_loss_weight) discourages router collapse onto a single expert.

Adds a sinusoidal(σ)→E router bias plus hard σ-band specialization: specialize_experts_by_sigma_buckets = true partitions the expert pool by timestep bucket so each σ-band only routes to its assigned experts (e.g. with num_experts = 6 and 3 buckets, 2 experts per band). sigma_bucket_boundaries = [0.0, 0.5, 0.8, 1.0] places the splits unevenly — more capacity in the late, low-noise refinement regime.

Variants
• hydralora — default 16GB+ profile.
• hydralora-8gb — same architecture, low-VRAM profile (gradient checkpointing + offload); pick this if you OOM on the default.

Training produces a *_moe.safetensors sibling next to the adapter; both files are needed for router-live inference (make test-hydra / ComfyUI Anima Adapter Loader). Requires cache_llm_adapter_outputs = true.
