FeRA

Author-faithful FeRA (Yin et al., arXiv:2511.17979). Independent-A stacked experts — each expert owns its own lora_down/lora_up — gated by a single global router fed by a frequency-energy index FEI(z_t) rather than the per-layer activation router used by HydraLoRA. Lives on the LoRA-family network module via the three-axis cfg use_moe_style="independent_A" + route_per_layer=false + router_source="fei".

Each expert is OrthoLoRA-parameterized (use_ortho=true) so per-expert (down, up) factors share PSOFT-style SVD bases with per-expert Cayley rotation + diagonal λ. FECL (frequency-energy consistency loss) is wired but defaults off — activating it doubles per-step cost via a second no-grad DiT forward.

Bench notes. The paper's 3-band split collapses to ~2 bands on Anima flow-matching latents (mid band <8% energy). Default keeps fera_num_bands=3 for author fidelity; flip to 2 with fera_fecl_weight=0 for the Anima-validated split. The σ_low = min(H_lat, W_lat) / fei_sigma_low_div rule (default div=4) is bench-validated for Anima — the paper's pixel-domain min(H, W)/128 is SD2-512²-specific.

Training produces anima_fera*.safetensors + a *_moe.safetensors sibling holding the global router; router-live inference uses make test-hydra / ComfyUI Anima Adapter Loader (the same sniff path as HydraLoRA). Requires cache_llm_adapter_outputs=true.
