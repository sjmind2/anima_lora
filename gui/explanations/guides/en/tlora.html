T-LoRA

Timestep-dependent rank masking. The effective LoRA rank varies with the denoising timestep via a power-law schedule:
• High noise (early steps) → full rank (maximum expressiveness)
• Low noise (late steps) → reduced rank (down to min_rank)
Concentrates capacity where structure is being decided. The mask is training-only — inference uses full rank at every t by design — so the saved .safetensors is a regular bakeable LoRA. See docs/methods/timestep_mask.md.

Toggle: use_timestep_mask = true.

Variants
• tlora — T-LoRA stacked on OrthoLoRA (use_ortho = true + use_timestep_mask = true). Default mergeable combo — rank-by-σ scheduling with orthogonal regularization. Fully bakeable.
• tlora-8gb — same architecture, low-VRAM profile (gradient checkpointing + offload); pick this if you OOM on the default.
