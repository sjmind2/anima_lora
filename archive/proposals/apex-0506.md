# APEX feasibility verdict — 2026-05-06

Status: **proceed, after one config change.** The previously-suspected
structural failure ("affine condition shift is invisible to RMSNorm
cross-attention on Anima") is **not** what is killing APEX. The shipped
init is wasteful but not dead; with `init_b = 0` the perturbation cleanly
clears every visibility gate.

This memo documents the bench-driven investigation that produced that
verdict, so that a future run can reproduce or extend it.

## TL;DR

- Default config (`apex_condition_shift_init_a = -0.5`,
  `apex_condition_shift_init_b = 1.0`) wastes ~97% of its perturbation
  budget on a softmax-invisible token-independent direction. It still
  delivers ~0.53 nats of attention-map divergence on real q's, but the
  gradient signal driving `(a, b)` is mostly tuning noise.
- **Change `apex_condition_shift_init_b` from `1.0` to `0.0`.** With
  `init_b = 0`, the perturbation collapses to a per-channel rescale that
  RMSNorm pipes through as a sign-flip on K. Real attention maps diverge
  by ~2.1 nats — 4× the gate, 4× the shipped config — with only 13%
  of the perturbation in the softmax-null subspace.
- Do **not** fork `ConditionShift` to per-token / K-V-rotation
  parametrizations yet. The affine family's sign-flip mode is
  structurally adequate.
- Pure rotation (`A = Q`, `b = 0`) also passes (~0.71 nats) but is
  dominated by sign-flip. Mentioned only because the deep-research
  report flagged it as a candidate fork.
- `--mode full` with default init (`A = -0.5·I`) is bit-identical to
  diag at uniform init. To test off-diagonal mixing you need
  `--full-orth-jitter > 0` (added in this round).

The earlier "97% softmax-invisible" finding was real, but it was
underestimating the impact of the visible 3% on trained q's. Real q's
align with K(c) in the directions kv_proj actually projects energy to —
small visible perturbations there produce disproportionate attention
divergence. The fix is still to stop wasting the other 97%, but the
parametrization itself doesn't need replacing.

## What we asked

1. Is APEX's `c_fake = A·c + b` strong enough to act as an endogenous
   adversary on Anima's RMSNorm + softmax cross-attention path?
2. If shipped scalar/diag is dead at Anima scale (config comment in
   `configs/methods/apex.toml` says `(a,b)` drifted <3% in 7k steps),
   is there a config / parametrization change that revives it, or is
   the method structurally mismatched?
3. Kill criterion: a cheap forward-pass probe whose verdict can stop a
   training run before its compute is spent.

## Bench design

`bench/apex/probe_attention_visibility.py` runs cached `crossattn_emb_v0`
and `c_fake = ConditionShift(c)` through every block's
`kv_proj → split → k_norm / v_norm`, optionally captures real q's via a
forward hook on `cross_attn.q_norm`, and reports five metrics:

| Metric                        | Role          | Default gate |
|-------------------------------|---------------|--------------|
| `attn_sym_kl`                 | primary       | median ≥ 0.5 nats |
| `k_pre_token_indep_frac`      | primary       | median ≤ 0.5 |
| `k_post_rel`                  | sanity floor  | median ≥ 0.05 |
| `v_post_rel` / `k_pre_rel`    | reported      | — |

Earlier iterations gated only on `k_post_rel`, which gave false PASS on
every shipped setting (large raw delta in K, but mostly in softmax's
null space). The two-gate rule directly measures the property APEX
needs (attention maps differ) plus the property that predicts
gradient-signal efficiency (perturbation lives in a softmax-visible
direction).

`--use-real-q` is the canonical mode. Real q's at `t=0.5` are captured
by hooks on `cross_attn.q_norm` during a single DiT forward at
`latent_hw=64` (32×32 patches, 1024 image tokens). Under random
Gaussian q's the bench is conservative — it underestimates `attn_sym_kl`
by roughly 8× because random q's have no preferential alignment with
K(c). Always use `--use-real-q`.

Reproducibility: `--seed` controls prompt sampling, q-slab generation,
random orthogonal init for `--full-orth-jitter`, and the captured q's
sampling indices. Same args produce bit-identical metrics.

## Runs

All runs against `output/ckpt/anima-tlora-0430-12.safetensors` warm-start
+ `models/diffusion_models/anima-preview3-base.safetensors`, 16 cached
prompts, 28 blocks, `--seed 0`. Raw artifacts under
`bench/apex/results/<YYYYMMDD-HHMM>-<label>/`.

### Synthetic q (conservative)

| Variant                                   | k_post_rel | indep_frac | attn_kl |
|-------------------------------------------|-----------:|-----------:|--------:|
| Shipped scalar (`a=-0.5, b=1`)            |       1.91 |       0.97 |   0.063 |
| Shipped diag, no warmstart                |       1.91 |       0.97 |   0.062 |
| Shipped diag, +warmstart                  |       1.91 |       0.97 |   0.063 |
| Full mode, default init (`A=-0.5·I, b=1`) |       1.91 |       0.97 |   0.063 |
| Sign-flip (`a=-0.5, b=0`)                 |       2.00 |       0.13 |   0.207 |
| Sign-flip + orth jitter 0.5               |       1.90 |       0.13 |   0.190 |
| Pure rotation (`A=Q, b=0`)                |       1.36 |       0.11 |   0.101 |

Every shipped variant collapses to the same number because uniform
diag (`a_i = -0.5 ∀i`) and `A = init_a · I` are mathematically
identical to scalar at init — only training drift breaks the
equivalence, and the report's claim that scalar `(a,b)` drift <3% in
7k steps suggests that's a real obstacle. Warm-start LoRA is loaded
correctly (840 keys including all four cross-attn projections × 28
blocks) but `‖ΔK‖/‖K‖` is dominated by base `kv_proj` geometry, so the
~1.5% LoRA delta on top doesn't shift the metrics visibly.

### Real q (canonical)

| Variant                          | k_post_rel | indep_frac | attn_kl | Verdict |
|----------------------------------|-----------:|-----------:|--------:|---------|
| Sanity `(a=+0.5, b=0)`           |      0.003 |       0.13 |   1e-5  | FAIL (k_post + KL near zero — RMSNorm fully absorbs +α scale) |
| Shipped scalar (`a=-0.5, b=1`)   |       1.91 |   **0.97** |    0.53 | FAIL on indep_frac |
| Shipped diag (`a=-0.5, b=1`)     |       1.91 |   **0.97** |    0.53 | FAIL on indep_frac |
| Pure rotation (`A=Q, b=0`)       |       1.36 |       0.11 |    0.71 | PASS |
| **Sign-flip (`a=-0.5, b=0`)**    |   **2.00** |   **0.13** | **2.09** | **PASS** |

The sanity row is what we expect from RMSNorm: positive scaling is
exactly absorbed (KL ~ 1e-5, k_post_rel ~ 3e-3). The bench correctly
flags this as a "wiring bug or zero perturbation" on the k_post sanity
floor — calibration is sound.

## Interpretation

`c_fake = A·c + b` on RMSNorm cross-attn has only a few attention-
altering directions:

1. **Positive diagonal** (`α > 0, b = 0`): post-norm K identical → 0
   KL. Sanity row confirms.
2. **Negative diagonal** (`α < 0, b = 0`): post-norm K exactly negated.
   Real q's that aligned with K(c) for high attention now anti-align
   with K(c_fake) → attention map "inverts". Real-q KL ≈ 2 nats.
3. **Off-diagonal** (`A = Q`): K vectors land in unrelated subspace,
   trained q's overlap drops uniformly → softmax flattens. Real-q KL
   ≈ 0.7 nats — visible but weaker than sign-flip.
4. **Bias `b ≠ 0`**: token-independent K-shift. Mostly absorbed by
   softmax (97% under uniform b). Drives gradient on `b` into mostly
   useless directions; the visible 3% nonetheless gives ≈ 0.5 nats KL
   on real q's because trained q's amplify the visible component.

This explains the failure mode the deep-research report predicted
("perturbation will degenerate into anchored self-distillation under
any hyperparameter sweep"): if `b` carries most of the perturbation
energy and the optimizer wastes its `(a, b)` updates tuning the bias,
training-time KL stalls at the 0.5-nat floor and never grows. With
`b = 0`, all the gradient on `a` goes into actively useful directions,
and the sign-flip ceiling at ~2 nats is the natural target.

The structural worry — "RMSNorm + softmax kill the perturbation" —
turns out to be a mis-aimed concern. RMSNorm kills positive scaling
and flattens norm differences, but it preserves the *sign* of K. Sign
information is exactly what the affine family can give. The
parametrization is fine; the init defaults aren't.

## What this changes about the recommendation

- **Don't fork `ConditionShift`.** Per-token learned offset / K-V
  subspace rotation / attention-KL-maximizing perturbation are all
  more flexible, but they add training-time cost and divergence risk
  while the existing parametrization can already deliver 2 nats of
  attention divergence on real q's. Save the fork for a fallback if
  the config change doesn't unstick training.
- **Don't pivot to teacher-based distillation yet.** The deep-research
  report's third recommendation (LCM-LoRA / Hyper-SD at 2-4 NFE) is
  still a fine plan B — but plan A no longer requires it.
- **Re-run APEX with `init_b = 0`.** This is a one-character config
  change (`apex_condition_shift_init_b = 0.0` in
  `configs/methods/apex.toml`).
- The 7k-step `(a, b)` drift problem documented in the apex.toml
  comment is consistent with the gradient-on-bias-is-mostly-useless
  story: with `b = 0`, the only trainable direction left is `a`,
  which has clean signal. Watch `apex/cs_a_norm` early in the next
  run — if it actually drifts away from -0.5 within the rampup
  window, we're past the "adversary stuck at init" symptom.

## What this does NOT prove

The bench rules out the *visibility* failure mode. It does **not**
rule out:

- **Warm-start coherence:** whether `x_fake = x_t − t·F_real` is a
  clean enough endpoint at small t under the warm-started base. The
  deep-research report flagged this as the second pillar; we haven't
  measured it. If it fails, sign-flip on `c_fake` won't matter — the
  fake branch fits noise.
- **Gradient quality of `L_fake`:** even with strong attention
  divergence on `c_fake`, the resulting `v_fake_sg` may or may not be
  a good correction signal for `T_mix`. APEX's self-confirming-fixed-
  point risk (jointly satisfied with itself but not better at
  sampling) is downstream of visibility.
- **Whether APEX clears NFE=4 baseline FM samples.** The deep-research
  report's recommended kill criterion is "NFE=4 quality on a fixed
  prompt set must beat baseline FM after rampup, otherwise stop."
  Bench passes don't predict this.

So the next gate, after the config change, is: run APEX with
`init_b = 0`, watch `apex/v_fake_divergence` and `apex/cs_a_norm` for
the first ~1k steps, and at the end of warmup compare NFE=4 samples
to baseline. The bench has done its job; from here, evidence comes
from training-time scalars.
